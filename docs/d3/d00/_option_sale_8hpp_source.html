29/**
30 * Option Sale event represents a trade or another market event with the price
31 * (for example, market open/close price, etc.) for each option symbol listed under the specified Underlying.
32 * Option Sales are intended to provide information about option trades <b>in a continuous time slice</b> with
33 * the additional metrics, like Option Volatility, Option Delta, and Underlying Price.
34 *
35 * <p>Option Sale events have unique @ref OptionSale::getIndex() "index" which can be used for later
36 * correction/cancellation processing.
37 *
38 * Option sale data source provides a consistent view of the set of known option sales.
39 * The corresponding information is carried in @ref OptionSale::getEventFlags() "eventFlags" property.
40 * The logic behind this property is detailed in IndexedEvent class documentation.
41 * Multiple event sources for the same symbol are not supported for option sale events, thus
42 * @ref OptionSale::getSource() "source" property is always @ref IndexedEventSource::DEFAULT "DEFAULT".
43 *
44 * This event is implemented on top of QDS records `OptionSale`.
45 */
46class DXFCPP_EXPORT OptionSale final : public MarketEvent, public IndexedEvent {
47 friend struct EventMapper;

59 * | | SM | | SS | SE | SB | RE | TX |
60 * +--------+----+----+----+----+----+----+----+

62 */

414 assert(sequence >= 0 && static_cast<std::uint32_t>(sequence) <= MAX_SEQUENCE);
