26/**
27 * Represents an extension of Order for the symbols traded on the OTC Markets. It includes the OTC Markets specific
28 * quote data, such as @ref OtcMarketsOrder::getQuoteAccessPayment() "Quote Access Payment (QAP) value",
29 * @ref OtcMarketsOrder::getOtcMarketsPriceType() "Quote Price Type",
30 * @ref OtcMarketsOrder::isOpen() "Open/Closed status",
31 * and @ref OtcMarketsOrder::isUnsolicited() "Unsolicited quote indicator". The original event (OTC Quote) published by
32 * the exchange is presented in the form of two separate OTCMarketsOrder events (bid and ask side).
33 * Note that the description of the fields (e.g. QAP) uses the original "quote" term.
34 * <p>
35 * For more information about original fields, QAP, Quote Flags and Extended Quote Flags,
36 * see <a href="https://downloads.dxfeed.com/specifications/OTC_Markets_Multicast_Data_Feeds.pdf">OTC Markets Multicast
37 * Data Feed</a>. <p> For more information about display requirements, see <a
38 * href="https://downloads.dxfeed.com/specifications/OTC_Markets_Data_Display_Requirements.pdf">OTC Markets Display
39 * Requirements</a>.
40 *
41 * <p> Like regular orders, OTC Markets order events arrive from multiple sources for the same market symbol and are
42 * distinguished by their @ref ::getIndex "index".
43 * It is unique across all the sources of depth information for the symbol.
44 * The event with @ref ::getSize() "size" either `0` or `NaN` is a signal to remove previously received order for the
45 * corresponding index.
46 * The method ::hasSize() is a convenient method to test for size presence.
47 *
48 * <h3>Quote Flags</h3>
49 *
50 * Quote Flags from OTC Markets are mapped to the access methods of this class as follows:
51 * <ul>
52 * <li>Update Side - @ref OtcMarketsOrder::getOrderSide() "orderSide"
53 * <li>State - @ref OtcMarketsOrder::isOpen() "open"
54 * <li>Ask/Bid Unsolicited - @ref OtcMarketsOrder::isUnsolicited() "unsolicited"
55 * <li>Ask/Bid Priced, Aks/Bid OW/BW - @ref OtcMarketsOrder::getOtcMarketsPriceType() "otcMarketsPriceType"
56 * </ul>
57 *
58 * <h3>Extended Quote Flags</h3>
59 *
60 * Extended Quote Flags from OTC Markets are mapped to the access methods of this class as follows:
61 * <ul>
62 * <li>QuoteSaturatedFlag - @ref OtcMarketsOrder::isSaturated() "saturated"
63 * <li>Bid/Offer AutoExFlag - @ref OtcMarketsOrder::isAutoExecution() "autoExecution"
64 * <li>NMSConditionalQuoteFlag - @ref OtcMarketsOrder::isNmsConditional() "nmsConditional"
65 * </ul>
66 *
67 * <h3><a name="eventFlagsSection">Event flags, transactions and snapshots</a></h3>
68 *
69 * OTC Markets order event sources provide a consistent view of the order book. Their updates
70 * may incorporate multiple changes to individual orders that have to be processed at the same time.
71 * The corresponding information is carried in @ref ::getEventFlags() "eventFlags" property.
72 *
73 * <p> See the "Event Flags" section of OrderBase class documentation for details.
74 *
75 * <h3>Publishing order books</h3>
76 *
77 * When publishing an order event with DXPublisher::publishEvents() method, least significant 32 bits of
78 * order @ref ::getIndex() "index" must be in a range of from 0 to `std::numeric_limits<std::int32_t>::max()`
79 * inclusive. Use ::setSource() method after ::setIndex() to properly include source identifier into the
80 * index.
81 *
82 * A snapshot has to be published in the <em>descending</em> order of @ref ::getIndex() "index", starting with
83 * an event with the largest index and marking it with EventFlag::SNAPSHOT_BEGIN bit in @ref ::getEventFlags()
84 * "eventFlags", and finishing the snapshot with an event that has zero 32 least significant bits of index.
85 * EventFlag::SNAPSHOT_END bit in @ref ::getEventFlags() "eventFlags" is optional during publishing.
86 * It will be properly set on receiving end anyway.
87 *
88 * <h3>Limitations</h3>
89 *
90 * This event type cannot be used with DXFeed::getLastEvent() method.
91 *
92 * <h3><a name="fobSection">Full Order Book Support</a></h3>
93 *
94 * Some feeds provide support for "Full Order Book" (FOB) where additional fields will be available:
95 * <ul>
96 * <li>@ref ::getAction() "action" - event business meaning (see OrderAction for more details)</li>
97 * <li>@ref ::getActionTime() "actionTime" - time of the last action</li>
98 * <li>@ref ::getOrderId() "orderId" - ID of this OTC Markets order</li>
99 * <li>@ref ::getAuxOrderId() "auxOrderId" - additional ID for this OTC Markets order</li>
100 * <li>@ref ::getTradeId() "tradeId" - trade (order execution) ID</li>
101 * <li>@ref ::getTradePrice() "tradePrice" - price of the trade</li>
102 * <li>@ref ::getTradeSize() "tradeSize" - size of the trade</li>
103 * </ul>
104 *
105 * <h3>Implementation details</h3>
106 *
107 * This event is implemented on top of QDS records `OtcMarketsOrder#<source-id>`,
108 * where `<source-id>` is up to 4 ASCII characters with a mnemonic for the source like "pink".
109 */
