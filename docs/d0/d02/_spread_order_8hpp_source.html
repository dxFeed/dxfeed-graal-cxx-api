23/**
24 * Spread order event is a snapshot for a full available market depth for all spreads
25 * on a given underlying symbol. The collection of spread order events of a symbol
26 * represents the most recent information that is available about spread orders on
27 * the market at any given moment of time.
28 *
29 * <p> Spread order is similar to a regular Order, but it has a
30 * @ref SpreadOrder::getSpreadSymbol() "spreadSymbol" property that contains the symbol
31 * of the actual spread that is being represented by spread order object.
32 * @ref SpreadOrder::getEventSymbol() "eventSymbol" property contains the underlying symbol
33 * that was used in subscription.
34 *
35 * <p> Like regular orders, spread order events arrive from
36 * multiple sources for the same market symbol and are distinguished by their
37 * @ref SpreadOrder::getIndex "index".
38 * It is unique across all the sources of depth information for the symbol.
39 * The event with @ref SpreadOrder::getSize() "size" either `0` or `NaN`
40 * is a signal to remove previously received order for the corresponding index.
41 * The method @ref SpreadOrder::hasSize() "hasSize" is a convenient method to test for size presence.
42 *
43 * <h3><a name="eventFlagsSection">Event flags, transactions and snapshots</a></h3>
44 *
45 * Some spread order event sources provide a consistent view of the price-level or detailed order book. Their updates
46 * may incorporate multiple changes to price levels or to individual orders that have to be processed at the same time.
47 * The corresponding information is carried in @ref SpreadOrder::getEventFlags() "eventFlags" property.
48 *
49 * <p> See `Event Flags` section of OrderBase class documentation for details.
50 *
51 * <p>IndexedEventModel class handles all the snapshot and transaction logic and conveniently represents
52 * a list current of events.
53 * It relies on the code of AbstractIndexedEventModel to handle this logic.
54 * Use the source code of AbstractIndexedEventModel for clarification on transactions and snapshot logic.
55 *
56 * <h3>Publishing order books</h3>
57 *
58 * When publishing an order event with DXPublisher::publishEvents() method, least significant 32 bits of
59 * order @ref SpreadOrder::getIndex() "index" must be in a range of from 0 to `std::numeric_limits<std::int32_t>::max()`
60 * inclusive. Use @ref SpreadOrder::setSource() "setSource" method after @ref SpreadOrder::setIndex() "setIndex" to
61 * properly include source identifier into the index.
62 *
63 * A snapshot has to be published in the <em>descending</em> order of @ref SpreadOrder::getIndex() "index", starting
64 * with an event with the largest index and marking it with SpreadOrder::SNAPSHOT_BEGIN bit in @ref
65 * SpreadOrder::getEventFlags() "eventFlags", and finishing the snapshot with an event that has zero 32 least
66 * significant bits of index.
67 * @ref SpreadOrder::SNAPSHOT_END bit in @ref SpreadOrder::getEventFlags() "eventFlags" is optional during publishing.
68 * It will be properly set on receiving end anyway.
69 *
70 * <h3>Limitations</h3>
71 *
72 * This event type cannot be used with @ref DXFeed::getLastEvent() method.
73 *
74 * <h3><a name="fobSection">Full Order Book Support</a></h3>
75 *
76 * Some feeds provide support for "Full Order Book" (FOB) where additional fields will be available:
77 * <ul>
78 * <li>@ref SpreadOrder::getAction() "action" - event business meaning (see OrderAction for more details)</li>
79 * <li>@ref SpreadOrder::getActionTime() "actionTime" - time of the last action</li>
80 * <li>@ref SpreadOrder::getOrderId() "orderId" - ID of this order</li>
81 * <li>@ref SpreadOrder::getAuxOrderId() "auxOrderId" - additional ID for this order</li>
82 * <li>@ref SpreadOrder::getTradeId() "tradeId" - trade (order execution) ID</li>
83 * <li>@ref SpreadOrder::getTradePrice() "tradePrice" - price of the trade</li>
84 * <li>@ref SpreadOrder::getTradeSize() "tradeSize" - size of the trade</li>
85 * </ul>
86 *
87 * <h3>Implementation details</h3>
88 *
89 * This event is implemented on top of QDS records `SpreadOrder#<source-id>`,
90 * where `<source-id>` is up to 4 ASCII characters with a mnemonic for the source like "ISE".
91 */
