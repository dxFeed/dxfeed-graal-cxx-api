22/**
23 * Provides on-demand historical tick data replay controls.
24 * This class is used to seamlessly transition from ordinary real-time or delayed data feed to the replay of the
25 * tick-by-tick history of market data behaviour without any changes to the code that consumes and process these market
26 * events.
27 *
28 * <p>As single OnDemandService instance is conceptually associated with each DXEndpoint endpoint instance with the role
29 * of @ref DXEndpoint::Role::FEED "FEED" or @ref DXEndpoint::Role::ON_DEMAND_FEED "ON_DEMAND_FEED".
30 * You can retrieve OnDemandService instance that is associated with a given
31 * DXEndpoint using @ref OnDemandService::getInstance(std::shared_ptr<DXEndpoint>) "getInstance(endpoint)" method.
32 *
33 * <p>For example, OnDemandService instance for a default DXFeed that is featured in the documentation of DXFeed class
34 * and is retrieved via
35 * @ref DXFeed::getInstance() "DXFeed::getInstance()" method can be acquired with the following code:
36 * <pre><tt>
37 * @ref DXEndpoint "auto" endpoint = DXEndpoint::getInstance();
38 * @ref OnDemandService "auto" onDemand = OnDemandService::@ref OnDemandService::getInstance(std::shared_ptr<DXEndpoint>) "getInstance"(endpoint);
39 * </tt></pre>
40 *
41 * This instance can be used for on-demand historical tick data replay only when endpoint is connected to
42 * the on-demand data provider and the appropriate credentials are provided.
43 * @ref OnDemandService::isReplaySupported() "isReplaySupported" returns `true` when it is so.
44 *
45 * <p>On-demand replay is started with @ref OnDemandService::replay(std::int64_t) "replay" method that takes time as a parameter.
46 * DXFeed is then disconnected from other data providers, pre-buffers historical tick data for replay,
47 * and starts replay of the data as if it was coming from the data feeds now. All the usual APIs that are
48 * part of DXFeed like @ref DXFeedSubscription "subscription" and various dxFeed models
49 * can be used normally as with the ordinary real-time or delayed data feed.
50 *
51 * <p>Replay speed can be changed on the fly with @ref OnDemandService::setSpeed(double) "setSpeed" method or it can be set
52 * initially when starting replay with a two-argument version of @ref OnDemandService::replay(std::int64_t, double) "replay" method.
53 * @ref OnDemandService::pause() "pause" method is the same as @ref OnDemandService::setSpeed(double) "setSpeed(0)".
54 *
55 * <p>@ref OnDemandService::stopAndResume() "stopAndResume" method stops data replay and resumes ordinary real-time or delayed data feed
56 * that was used before @ref OnDemandService::replay(std::int64_t) "replay" method was invoked.
57 * Endpoints with a role of
58 * @ref DXEndpoint::Role::ON_DEMAND_FEED "ON_DEMAND_FEED" do not have an ordinary feed (they are essentially on-demand only)
59 * and thus @ref OnDemandService::stopAndResume() "stopAndResume" method works like @ref OnDemandService::stopAndClear() "stopAndClear" for them.
60 *
61 * <h3>State and change notifications</h3>
62 *
63 * On-demand historical tick data replay state can be queried with @ref OnDemandService::isReplaySupported() "isReplaySupported",
64 * @ref OnDemandService::isReplay() "isReplay", @ref OnDemandService::getSpeed() "getSpeed", and @ref OnDemandService::getTime() "getTime" methods.
65 *
66 * <h3>Threads and locks</h3>
67 *
68 * This class is thread-safe and can be used concurrently from multiple threads without external synchronization.
69 */

123 /**

173 /**
174 * Turns on-demand historical data replay mode from a specified time with real-time speed.
175 * This is a shortcut for:
176 * <pre><tt>
177 * @ref OnDemandService::replay(Date, double) "replay"(time, 1);</tt></pre>
178 * This method can be used only when OnDemandService::isReplaySupported() method returns `true`,

183 * @param time time (timestamp in millis) to start replay from.
184 */
185 void replay(std::int64_t time) const noexcept;

187 /**
188 * Turns on-demand historical data replay mode from a specified time and with a specified speed.
189 * This method can be used only when OnDemandService::isReplaySupported() method returns `true`,
190 * that is when DXEndpoint is @ref DXEndpoint::connect() "connected"
191 * to an address with "(ondemand:<address>)" component that specifies an on-demand historical
192 * data provider address.
193 *
194 * <p>After invocation of this method:
195 * <ul>
196 * <li>OnDemandService::isReplay() returns `true`.
197 * <li>OnDemandService::isClear() returns `false`.
198 * <li>OnDemandService::getTime() returns time that is currently being replayed.
199 * <li>OnDemandService::getSpeed() returns `speed`.
200 * </ul>
201 *
202 * @param time time (timestamp in millis) to start replay from.
203 * @param speed speed to start replay with. Use 1 for real-time speed, >1 for faster than real-time speed,
204 * <1 for slower than real-time speed, and 0 for pause.
205 */
206 void replay(std::int64_t time, double speed) const noexcept;
207
208 /**
209 * Pauses on-demand historical data replay and keeps data snapshot.
210 * This method can only be called in @ref OnDemandService::isReplay() "replay mode".
211 * This is a shortcut for:
212 * <pre><tt>
213 * @ref OnDemandService::setSpeed(double) "setSpeed"(0);</tt></pre>
214 * This method atomically captures current replay time and pauses at it.
215 * After invocation of this method:
216 * <ul>
217 * <li>OnDemandService::isReplay() returns `true`.
218 * <li>OnDemandService::isClear() returns `false`.
219 * <li>OnDemandService::getTime() returns time where replay was paused at.
220 * <li>OnDemandService::getSpeed() returns `0`.
221 * </ul>
222 */
223 void pause() const noexcept;
224
225 /**
226 * Stops on-demand historical data replay and resumes ordinary data feed.
227 * This method has no effect when invoked not in @ref OnDemandService::isReplay() "replay mode".
228 * After invocation of this method:
229 * <ul>
230 * <li>OnDemandService::isReplay() returns `false`.
231 * <li>OnDemandService::isClear() returns `false`.
232 * <li>OnDemandService::getTime() returns last replayed time.
233 * <li>OnDemandService::getSpeed() returns `0`.
234 * </ul>
235 *
236 * <p>To stop on-demand historical data replay without resuming ordinary data feed use
237 * either OnDemandService::pause() to keep data snapshot or OnDemandService::stopAndClear() to clear data.
238 *
239 * <p>Note, that endpoints with a role of
240 * @ref DXEndpoint::Role::ON_DEMAND_FEED "ON_DEMAND_FEED" do not have an ordinary feed (they are essentially on-demand only)
241 * and thus `stopAndResume` method works like @ref OnDemandService::stopAndClear() "stopAndClear" for them.
242 */
243 void stopAndResume() const noexcept;
244
245 /**
246 * Stops incoming data and clears it without resuming data updates.
247 * This method works both in on-demand historical data replay mode and
248 * for ordinary data feed. All incoming data updates are suspended and current data is cleared.
249 * After invocation of this method:
250 * <ul>
251 * <li>OnDemandService::isReplay() returns `false`.
252 * <li>OnDemandService::isClear() returns `true`.
253 * <li>OnDemandService::getTime() returns last replayed time.
254 * <li>OnDemandService::getSpeed() returns `0`.
255 * </ul>
256 *
257 * <p>Ordinary data feed can be resumed with OnDemandService::stopAndResume() method and on-demand historical data
258 * replay can be continued with @ref OnDemandService::replay(std::int64_t, double) "replay(...)" method.
259 */
260 void stopAndClear() const noexcept;
261
262 /**
263 * Changes on-demand historical data replay speed while continuing replay at current @ref OnDemandService::getTime() "time".
264 * Speed is measured with respect to the real-time playback speed.
265 * This method can only be called with non-zero speed in @ref OnDemandService::isReplay() "replay mode".
266 *
267 * @param speed on-demand historical data replay speed.
268 */
269 void setSpeed(double speed) const noexcept;
270};
