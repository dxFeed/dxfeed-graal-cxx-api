10/**
11 * This marker interface marks subscription symbol classes (like TimeSeriesSubscriptionSymbol)
12 * that attach "filters" to the actual symbols. Filtered subscription symbols implement their `operator ==`
13 * and `std::hash<>` based on their symbol only, without considering their "filter" part (for example, a
14 * TimeSeriesSubscriptionSymbol has a @ref TimeSeriesSubscriptionSymbol::getFromTime() "fromTime" filter on a time range
15 * of events).
16 *
17 * <p>DXFeedSubscription has the following behavior for filtered symbols. There can be only one
18 * active filter per symbol. Adding new filtered symbol with the same symbol but different filter
19 * removes the old one from subscription, adds the new one, and <em>notifies</em>
20 * ObservableSubscriptionChangeListener <em>about this subscription change</em>.
21 * The later is a special behaviour for filtered subscription symbols, because on other types of
22 * symbols (like StringSymbol, for example) there is no notification when replacing one symbol with the other that
23 * is "equal" to the first one.
24 */
25struct FilteredSubscriptionSymbol {};
