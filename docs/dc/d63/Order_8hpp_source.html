22 /**
23  * Order event is a snapshot for a full available market depth for a symbol.
24  * The collection of order events of a symbol represents the most recent information
25  * that is available about orders on the market at any given moment of time.
26  * Order events give information on several levels of details, called scopes - see Scope.
27  * Scope of an order is available via @ref Order::getScope "scope" property.
28  *
29  * <p> Order events arrive from
30  * multiple sources for the same market symbol and are distinguished by their @ref Order::getIndex "index". Index is a
31  * unique per symbol identifier of the event.
32  * It is unique across all the sources of depth information for the symbol.
33  * The event with @ref Order::getSize() "size" either `0` or `NaN` is a signal to remove previously received order for the
34  * corresponding index.
35  * The method Order::hasSize() is a convenient method to test for size presence.
36  *
37  * <p> Events from finer-grained Scope of detail give more information and include events
38  * from coarse-grained Scope of detail. For a consistent representation of the market depth
39  * to the end-user, order events must be filtered to remove coarse-grained events that
40  * are superseded by the finer-grained events.
41  *
42  * <h3><a name="eventFlagsSection">Event flags, transactions and snapshots</a></h3>
43  *
44  * Some order event sources provide a consistent view of the price-level or detailed order book. Their updates
45  * may incorporate multiple changes to price levels or to individual orders that have to be processed at the same time.
46  * The corresponding information is carried in @ref Order::getEventFlags() "eventFlags" property.
47  *
48  * <p> See `Event Flags` section of OrderBase class documentation for details.
49  *
50  * <p> The composite quotes with Scope::COMPOSITE and regional quotes with Scope::REGIONAL come
51  * individually from different venues and are not related to each other in any transactional way. The result of
52  * Order::getEventFlags method for them is always zero.
53  *
54  * <h3>Publishing order books</h3>
55  *
56  * When publishing an order event with DXPublisher::publishEvents() method, least significant 32 bits of
57  * order @ref Order::getIndex() "index" must be in a range of from 0 to `std::numeric_limits<std::int32_t>::max()` inclusive.
58  * Use Order::setSource() method after Order::setIndex() to properly include source identifier into the index.
59  *
60  * A snapshot has to be published in the <em>descending</em> order of @ref Order::getIndex() "index", starting with
61  * an event with the largest index and marking it with EventFlag::SNAPSHOT_BEGIN bit in @ref Order::getEventFlags() "eventFlags",
62  * and finishing the snapshot with an event that has zero 32 least significant bits of index.
63  * EventFlag::SNAPSHOT_END bit in @ref Order::getEventFlags() "eventFlags" is optional during publishing.
64  * It will be properly set on receiving end anyway.
65  *
66  * <h3>Limitations</h3>
67  *
68  * This event type cannot be used with DXFeed::getLastEvent() method.
69  *
70  * <h3><a name="fobSection">Full Order Book Support</a></h3>
71  *
72  * Some feeds provide support for "Full Order Book" (FOB) where additional fields will be available:
73  * <ul>
74  * <li>@ref Order::getAction() "action" - event business meaning (see OrderAction for more details)</li>
75  * <li>@ref Order::getActionTime() "actionTime" - time of the last action</li>
76  * <li>@ref Order::getOrderId() "orderId" - ID of this order</li>
77  * <li>@ref Order::getAuxOrderId() "auxOrderId" - additional ID for this order</li>
78  * <li>@ref Order::getTradeId() "tradeId" - trade (order execution) ID</li>
79  * <li>@ref Order::getTradePrice() "tradePrice" - price of the trade</li>
80  * <li>@ref Order::getTradeSize() "tradeSize" - size of the trade</li>
81  * </ul>
82  *
83  * <h3>Implementation details</h3>
84  *
85  * This event is implemented on top of QDS record `Quote` for composite quotes with Scope::COMPOSITE,
86  * record `Quote&X` for regional exchange best quotes with Scope::REGIONAL,
87  * record `MarketMaker` for market-maker quotes and futures price level aggregates with Scope::AGGREGATE,
88  * record `Order` for the most fine-grained Scope::ORDER with zero source id,
89  * and records `Order#<source-id>` for specific source ids,
90  * where `<source-id>` is up to 4 ASCII characters with a mnemonic for the source like "NTV".
91  */
