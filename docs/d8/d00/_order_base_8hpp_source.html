27/**
28 * Base class for common fields of Order, AnalyticOrder and SpreadOrder events.
29 * Order events represent a snapshot for a full available market depth for a symbol.
30 * The collection of order events of a symbol represents the most recent information that is
31 * available about orders on the market at any given moment of time.
32 *
33 * <p>Order event represents market depth for a <b>specific symbol</b>.
34 *
35 * <p>AnalyticOrder event represents market depth for a <b>specific symbol</b> extended with an analytic
36 * information, for example, whether particular order represent an iceberg or not.
37 *
38 * <p>SpreadOrder event represents market depth for
39 * <b>all spreads on a given underlying symbol</b>.
40 *
41 * <p> Order events arrive from
42 * multiple sources for the same market symbol and are distinguished by their
43 * @ref OrderBase::getIndex "index". Index is a unique per symbol identifier of the event.
44 * It is unique across all the sources of depth information for the symbol.
45 * The event with @ref OrderBase::getSize() "size" either `0` or `NaN` is a signal to remove previously received order
46 * for the corresponding index.
47 * The method @ref OrderBase::hasSize() "hasSize" is a convenient method to test for size presence.
48 *
49 * <h3><a name="eventFlagsSection">Event flags, transactions and snapshots</a></h3>
50 *
51 * Some order event sources provide a consistent view of the price-level or detailed order book. Their updates
52 * may incorporate multiple changes to price levels or to individual orders that have to be processed at the same time.
53 * The corresponding information is carried in @ref OrderBase::getEventFlags() "eventFlags" property.
54 * The logic behind this property is detailed in IndexedEvent class documentation.
55 *
56 * <p> The event @ref OrderBase::getSource() "source" identifier for an order is a part of the unique event @ref
57 * OrderBase::getIndex() "index". It occupies highest bits of the @ref OrderBase::getIndex() "index" (index is
58 * not-negative). The lowest bits of
59 * @ref OrderBase::getIndex() "index" contain source-specific event index which is always zero in
60 * an event that is marked with EventFlag::SNAPSHOT_END bit in @ref OrderBase::getEventFlags() "eventFlags".
61 *
62 * <p> Note that for an order with EventFlag::REMOVE_EVENT bit in @ref OrderBase::getEventFlags() "eventFlags"
63 * it is always the case that @ref #getSize() "size" is either `0` or `NaN`,
64 * so no additional logic to process this bit is required for orders.
65 * Transactions and snapshots may include orders with @ref OrderBase::getSize() "size" of `0`.
66 * The filtering that distinguishes those events as removals of orders shall be performed after
67 * all transactions and snapshot processing.
68 *
69 * <p> Some aggregated feeds (like CME market depth) are mapped into two distinct source ids (one for
70 * buy orders and one for sell orders), but updates and transactions may span both. It is important to keep a
71 * separate track of transactional state for each source id, but, at the same time, when
72 * DXFeedEventListener::eventsReceived() method is called for a list
73 * of events, the order book shall be considered complete and consistent only when all events from the given
74 * list are processed.
75 */

88 * Flags property has several significant bits that are packed into an integer in the following way:

111 /*
112 * Index field contains source identifier, optional exchange code and low-end index (virtual id or MMID).
113 * Index field has 2 formats depending on whether source is "special" (see OrderSource.isSpecialSourceId()).
114 * Note: both formats are IMPLEMENTATION DETAILS, they are subject to change without notice.
115 * 63..48 47..32 31..16 15..0
116 * +--------+--------+--------+--------+
117 * | Source |Exchange| Index | <- "special" order sources (non-printable id with exchange)
118 * +--------+--------+--------+--------+
119 * 63..48 47..32 31..16 15..0
120 * +--------+--------+--------+--------+
121 * | Source | Index | <- generic order sources (alphanumeric id without exchange)
122 * +--------+--------+--------+--------+
123 * Note: when modifying formats track usages of getIndex/setIndex, getSource/setSource and isSpecialSourceId
124 * methods in order to find and modify all code dependent on current formats.
125 */

131 * | | SM | | SS | SE | SB | RE | TX |
132 * +---------+----+----+----+----+----+----+----+
133 */

177 /**
178 * Returns source of this event.
179 * The source is stored in the highest bits of the @ref getIndex() "index" of this event.
180 *
181 * @return source of this event.
182 */
183 const OrderSource &getSource() const & noexcept override {
184 auto sourceId = static_cast<std::int32_t>(sar(orderBaseData_.index, SPECIAL_SOURCE_ID_SHIFT));
185
186 if (!OrderSource::isSpecialSourceId(sourceId)) {
187 sourceId = static_cast<std::int32_t>(sar(orderBaseData_.index, NONSPECIAL_SOURCE_ID_SHIFT));
188 }
189
190 return OrderSource::valueOf(sourceId);
191 }

193 /**
194 * Changes source of this event.
195 * This method changes highest bits of the @ref OrderBase::getIndex() "index" of this event.
196 *
197 * @param source source of this event.
198 */
199 void setSource(const OrderSource &source) noexcept {
200 auto shift = OrderSource::isSpecialSourceId(source.id()) ? SPECIAL_SOURCE_ID_SHIFT : NONSPECIAL_SOURCE_ID_SHIFT;
201 std::int64_t mask = OrderSource::isSpecialSourceId(
202 static_cast<std::int32_t>(sar(orderBaseData_.index, SPECIAL_SOURCE_ID_SHIFT)))
203 ? ~sal(std::int64_t{-1}, SPECIAL_SOURCE_ID_SHIFT)
204 : ~sal(std::int64_t{-1}, NONSPECIAL_SOURCE_ID_SHIFT);
205 orderBaseData_.index =
206 andOp(sal(static_cast<std::int64_t>(source.id()), shift), andOp(orderBaseData_.index, mask));
207 }
