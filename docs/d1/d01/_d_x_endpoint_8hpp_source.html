31/**
32 * Manages network connections to @ref DXFeed "feed" or
33 * @ref DXPublisher "publisher". There are per-process (per GraalVM Isolate for now) ready-to-use singleton instances
34 * that are available with ::getInstance() and ::getInstance(Role) methods as well as
35 * factory methods ::create() and ::create(Role), and a number of configuration methods. Advanced
36 * properties can be configured using
37 * @ref ::newBuilder() "newBuilder()".@ref Builder#withProperty(const std::string&, const std::string&)
38 * "withProperty(key, value)".@ref Builder::build() "build()".
39 *
40 * See DXFeed for details on how to subscribe to symbols and receive events.
41 *
42 * <h3>Endpoint role</h3>
43 *
44 * Each endpoint has a role that is specified on its creation and cannot be changed afterwards.
45 * The default factory method ::create() creates an endpoint with a @ref Role::FEED "FEED" role.
46 * Endpoints with other roles are created with ::create(Role) factory method. Endpoint role is
47 * represented by @ref Role "DXEndpoint::Role" enumeration.
48 *
49 * Endpoint role defines the behavior of its @ref #connect(const std::string&) "connect" method:
50 *
51 * - @ref Role::FEED "FEED" connects to the remote data feed provider and is optimized for real-time or
52 * delayed data processing (<b>this is a default role</b>).
53 * ::getFeed() method returns a feed object that subscribes to this remote data feed provider and receives events
54 * from it. When event processing threads cannot keep up (don't have enough CPU time), data is dynamically conflated
55 * to minimize latency between received events and their processing time.
56 * For example:
57 * - <b>`DXEndpoint::create()->connect("demo.dxfeed.com:7300")->getFeed()`</b> returns a demo feed from dxFeed with
58 * sample market quotes.
59 * - <b>`DXEndpoint::create()->connect("localhost:7400")->getFeed()`</b> returns a feed that is connected to a
60 * publisher that is running on the same host. See example below.
61 * - <b>`DXEndpoint::create()->connect("file:demo-sample.data")->getFeed()`</b> returns a feed that is connected to
62 * a "demo-sample.data" file and plays back it as if it was received in real time.
63 *
64 * This endpoint is automatically connected to the configured data feed as explained in
65 * <a href="#defaultPropertiesSection">default properties section</a>.
66 * - @ref Role::ON_DEMAND_FEED "ON_DEMAND_FEED" is similar to @ref Role::FEED "FEED", but it is designed to be used with
67 * OnDemandService for historical data replay only. It is configured with <a href="#defaultPropertiesSection">default
68 * properties</a>, but is not connected automatically to the data provider until @ref OnDemandService#replay(Date,
69 * double) "OnDemandService.replay" method is invoked.
70 * - @ref Role::STREAM_FEED "STREAM_FEED" is similar to @ref Role::FEED "FEED" and also connects to the remote data
71 * feed provider, but is designed for bulk parsing of data from files. DXEndpoint::getFeed() method returns feed
72 * object that subscribes to the data from the opened files and receives events from them. Events from the files are
73 * not conflated and are processed as fast as possible. Note, that in this role, DXFeed::getLastEvent method does not
74 * work and time-series subscription is not supported.
75 * For example:
76 * ```cpp
77 * auto endpoint = DXEndpoint::create(DXEndpoint::Role::STREAM_FEED);
78 * auto feed = endpoint->getFeed();
79 * ```
80 * creates a feed that is ready to read data from file as soon as the following code is invoked:
81 * ```cpp
82 * endpoint->connect("file:demo-sample.data[speed=max]");
83 * ```
84 * "[speed=max]" clause forces to the file reader to play back all the data from "demo-sample.data" file as fast as
85 * data subscribers are processing it.
86 * - @ref Role::PUBLISHER "PUBLISHER" connects to the remote publisher hub (also known as multiplexor) or creates a
87 * publisher on the local host. ::getPublisher() method returns a publisher object that publishes events to all
88 * connected feeds.
89 * For example: <b>`DXEndpoint->create(DXEndpoint::Role::PUBLISHER)->connect(":7400")->getPublisher()`</b>
90 * returns a publisher that is waiting for connections on TCP/IP port 7400. The published events will be delivered to
91 * all feeds that are connected to this publisher.
92 * This endpoint is automatically connected to the configured data feed as explained in <a
93 * href="#defaultPropertiesSection">default properties section</a>.
94 * - @ref Role::LOCAL_HUB "LOCAL_HUB" creates a local hub without ability to establish network connections.
95 * Events that are published via {@link #getPublisher() publisher} are delivered to local @ref ::getFeed() "feed"
96 * only.
97 *
98 * <h3>Endpoint state</h3>
99 *
100 * Each endpoint has a state that can be retrieved with ::getState() method.
101 * When endpoint is created with any role and default address is not specified in
102 * <a href="#defaultPropertiesSection">default properties</a>, then it is not connected to any remote endpoint.
103 * Its state is @ref State::NOT_CONNECTED "NOT_CONNECTED".
104 *
105 * @ref Role#FEED "Feed" and @ref Role#PUBLISHER "publisher" endpoints can connect to remote endpoints of the opposite
106 * role. Connection is initiated by @ref ::connect(const std::string&) "connect" method.
107 * The endpoint state becomes @ref State::CONNECTING "CONNECTING".
108 *
109 * When the actual connection to the remote endpoint is established, the endpoint state becomes
110 * @ref State::CONNECTED "CONNECTED".
111 *
112 * Network connections can temporarily break and return endpoint back into @ref State::CONNECTING "CONNECTING" state.
113 * File connections can be completed and return endpoint into @ref State::NOT_CONNECTED "NOT_CONNECTED" state.
114 *
115 * Connection to the remote endpoint can be terminated with ::disconnect() method.
116 * The endpoint state becomes @ref State::NOT_CONNECTED "NOT_CONNECTED".
117 *
118 * Endpoint can be closed with ::close() method. The endpoint state becomes @ref State::CLOSED "CLOSED". This is a
119 * final state. All connection are terminated and all internal resources that are held by this endpoint are freed.
120 * No further connections can be initiated.
121 *
122 * <h3>Event times</h3>
123 *
124 * The EventType::getEventTime() on received events is available only when the endpoint is created with
125 * ::DXENDPOINT_EVENT_TIME_PROPERTY property and the data source has embedded event times. This is typically true only
126 * for data events that are read from historical tape files (see above) and from OnDemandService.
127 * Events that are coming from a network connections do not have an embedded event time information and
128 * event time is not available for them anyway.
129 *
130 * <h3><a name="defaultPropertiesSection">Default properties</a></h3>
131 *
132 * Default properties are loaded from "dxfeed.properties" or "dxpublisher.properties" file depending on
133 * the @ref Role "role" of created endpoint. "dxfeed.properties" is used for
134 * @ref Role::FEED "FEED" and @ref Role::ON_DEMAND_FEED "ON_DEMAND_FEED",
135 * "dxpublisher.properties" is used for @ref Role::PUBLISHER "PUBLISHER"}.
136 * @ref Role::STREAM_FEED "STREAM_FEED" and @ref Role::LOCAL_HUB "LOCAL_HUB" do not support properties file.
137 *
138 * The location of this file can be specified using
139 * @ref Builder::withProperty(const std::string&, const std::string&) "withProperty"(::DXFEED_PROPERTIES_PROPERTY, path)
140 * or
141 * @ref Builder::withProperty(const std::string&, const std::string&) "withProperty"(::DXPUBLISHER_PROPERTIES_PROPERTY,
142 * path) correspondingly. When the location of this file is not explicitly specified using
143 * @ref Builder::withProperty(const std::string&, const std::string&) "withProperty" method, then the file path is taken
144 * from a system property with the corresponding name.
145 *
146 * When the path to the above properties file is not provided, then a resource named "dxfeed.properties" or
147 * "dxpublisher.properties" is loaded from classpath. When classpath is set to "." (current directory),
148 * it means that the corresponding file can be placed into the current directory with any need to specify additional
149 * properties.
150 *
151 * Defaults for individual properties can be also provided using system properties when they are not specified
152 * in the configuration file. System properties override configuration loaded from classpath resource, but don't
153 * override configuration from the user-specified configuration file.
154 *
155 * The ::NAME_PROPERTY is the exception from the above rule. It is never loaded from system properties.
156 * It can be only specified in configuration file or programmatically. There is a convenience
157 * @ref Builder::withName(const std::string&) "Builder.withName" method for it. It is recommended to assign short and
158 * meaningful endpoint names when multiple endpoints are used in the same process (one GraalVM Isolate for now).
159 * The name of the endpoint shall describe its role in the particular application.
160 *
161 * Note, that individual properties that are programmatically set using @ref Builder::withProperty(const std::string&,
162 * const std::string&) "withProperty" method always take precedence.
163 *
164 * @ref Role::FEED "FEED" and @ref Role::PUBLISHER "PUBLISHER" automatically establish connection on creation
165 * when the corresponding ::DXFEED_ADDRESS_PROPERTY or ::DXPUBLISHER_ADDRESS_PROPERTY is specified.
166 *
167 * <h3>Permanent subscription</h3>
168 *
169 * Endpoint properties can define permanent subscription for specific sets of symbols and event types in
170 * the data feed, so that DXFeed methods like @ref DXFeed::getLastEventIfSubscribed "getLastEventIfSubscribed",
171 * @ref DXFeed::getIndexedEventsIfSubscribed "getIndexedEventsIfSubscribed", and
172 * @ref DXFeed::getTimeSeriesIfSubscribed "getTimeSeriesIfSubscribed" can be used without a need to create a
173 * separate DXFeedSubscription object. Please, contact dxFeed support for details
174 * on the required configuration.
175 *
176 * <h3>Threads and locks</h3>
177 *
178 * This class is thread-safe and can be used concurrently from multiple threads without external synchronization.
179 *
180 * [Javadoc.](https://docs.dxfeed.com/dxfeed/api/com/dxfeed/api/DXEndpoint.html)
181 */

341 * delayed data processing (<b>this is a default role</b>). DXEndpoint::getFeed() method

357 *
358 * `ON_DEMAND_FEED` endpoint cannot be connected to an ordinary data feed at all.

361 * @see OnDemandService
