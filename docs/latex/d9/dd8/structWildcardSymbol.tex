\doxysubsubsection{Detailed Description}
Represents \mbox{[}wildcard\mbox{]} subscription to all events of the specific event type. 

The \mbox{\hyperlink{structWildcardSymbol_acf6b6b4eeecdb9cca6e311ed7db387f6}{Wildcard\+Symbol\+::\+ALL}} constant can be added to any \mbox{\hyperlink{classDXFeedSubscription}{DXFeed\+Subscription}} instance with \mbox{\hyperlink{classDXFeedSubscription_a43713365d3db41d8cf9e3a9e894611a4}{add\+Symbols}} method to the effect of subscribing to all possible event symbols. The corresponding subscription will start receiving all published events of the corresponding types.

Represents \mbox{[}wildcard\mbox{]} subscription to all events of the specific event type. 

{\bfseries{NOTE\+:}} Wildcard subscription can create extremely high network and CPU load for certain kinds of high-\/frequency events like quotes. It requires a special arrangement on the side of upstream data provider and is disabled by default in upstream feed configuration. Make that sure you have adequate resources and understand the impact before using it. It can be used for low-\/frequency events only (like Forex quotes), because each instance of \mbox{\hyperlink{classDXFeedSubscription}{DXFeed\+Subscription}} processes events in a single thread and there is no provision to load-\/balance wildcard subscription amongst multiple threads. Contact your data provider for the corresponding configuration arrangement if needed.
