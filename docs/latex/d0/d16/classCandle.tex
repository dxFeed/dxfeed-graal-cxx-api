\doxysubsubsection{Detailed Description}
\mbox{\hyperlink{classCandle}{Candle}} event with open, high, low, close prices and other information for a specific period. 

Candles are build with a specified \mbox{\hyperlink{structCandlePeriod}{Candle\+Period}} using a specified \mbox{\hyperlink{structCandlePrice}{Candle\+Price}} type with a data taken from the specified \mbox{\hyperlink{structCandleExchange}{Candle\+Exchange}} from the specified \mbox{\hyperlink{structCandleSession}{Candle\+Session}} with further details of aggregation provided by \mbox{\hyperlink{structCandleAlignment}{Candle\+Alignment}}.

Event symbol of the candle is represented with \mbox{\hyperlink{structCandleSymbol}{Candle\+Symbol}} class. Since the \`{}\+Candle is a time-\/series event, it is typically subscribed to using DXFeed\+Time\+Series\+Subscription class that handles the necessarily wrapping of the symbol into \mbox{\hyperlink{classTimeSeriesSubscriptionSymbol}{Time\+Series\+Subscription\+Symbol}} to specify a subscription time range.

\doxysubparagraph*{\label{_eventFlagsSection}%
Event flags, transactions and snapshots}

Some candle sources provide a consistent view of the set of known candles. The corresponding information is carried in \mbox{\hyperlink{classCandle_aec9692d1a464d7526280204185b6f5c5}{event\+Flags}} property. The logic behind this property is detailed in \mbox{\hyperlink{structIndexedEvent}{Indexed\+Event}} class documentation. Multiple event sources for the same symbol are not supported for candles, thus \mbox{\hyperlink{structTimeSeriesEvent_ababb3568ebcdb65d4fa1ed5811d4ac1a}{source}} property is always \mbox{\hyperlink{classIndexedEventSource_ab359b82d74ee151707106b66eec0a4e3}{DEFAULT}}.

Time\+Series\+Event\+Model class handles all the snapshot and transaction logic and conveniently represents a list current of time-\/series events order by their \mbox{\hyperlink{classCandle_a2a229f6b56a57a9277bac05388466ebc}{time}}. It relies on the code of Abstract\+Indexed\+Event\+Model to handle this logic. Use the source code of Abstract\+Indexed\+Event\+Model for clarification on transactions and snapshot logic.

\doxysubparagraph*{Publishing Candles}

Publishing of candle events follows the general rules explained in \mbox{\hyperlink{structTimeSeriesEvent}{Time\+Series\+Event}} class documentation.

\doxysubparagraph*{Implementation details}

This event is implemented on top of QDS record {\ttfamily Trade\+History} for tick candles with \mbox{\hyperlink{structCandlePeriod_a2934f6797a1c4aeb8ff49560370f0550}{Candle\+Period\+::\+TICK}}, records {\ttfamily \mbox{\hyperlink{classTrade}{Trade}}.$<$period$>$} for a certain set of popular periods, and QDS record {\ttfamily \mbox{\hyperlink{classCandle}{Candle}}} for arbitrary custom periods, with a set of {\ttfamily \mbox{\hyperlink{classCandle}{Candle}}\{$<$attributes$>$\}} records for a popular combinations of custom candle symbol attributes like \mbox{\hyperlink{structCandlePrice}{Candle\+Price}} for an efficient support of bid-\/ask charting.

Returns time when event was created or zero when time is not available. 

This event time is available only when the corresponding \mbox{\hyperlink{structDXEndpoint}{DXEndpoint}} is created with \mbox{\hyperlink{structDXEndpoint_a59bc3053ef198595c2ed5a450ba5a2a4}{DXENDPOINT\+\_\+\+EVENT\+\_\+\+TIME\+\_\+\+PROPERTY}} and the data source has embedded event times. This is typically true only for data events that are read from historical tape files and from \mbox{\hyperlink{structOnDemandService}{On\+Demand\+Service}}. Events that are coming from a network connections do not have an embedded event time information and this method will return zero for them, meaning that event was received just now.

Default implementation returns 0.
