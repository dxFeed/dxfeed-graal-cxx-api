\doxysubsubsection{Detailed Description}

Advanced properties can be configured using \doxylink{structdxfcpp_1_1_d_x_endpoint_a4fd9e984ae1cc206b6ccf9e7d528af29}{new\+Builder()}.\doxylink{classdxfcpp_1_1_d_x_endpoint_1_1_builder_a06466a9c8a88df62c32c20e17788f4a4}{with\+Property(key, value)}.\doxylink{classdxfcpp_1_1_d_x_endpoint_1_1_builder_a963b99c59a3721dab1472e47789e434f}{build()}.

See \doxylink{structdxfcpp_1_1_d_x_feed}{DXFeed} for details on how to subscribe to symbols and receive events.

\doxyparagraph*{Endpoint role}

Each endpoint has a role that is specified on its creation and cannot be changed afterwards. The default factory method \doxylink{structdxfcpp_1_1_d_x_endpoint_a4933409731472ca5aa39d0c83d542428}{DXEndpoint\+::create()} creates an endpoint with a \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2dae352a08d4fba713e5ae1e538bf27258c}{FEED} role. Endpoints with other roles are created with \doxylink{structdxfcpp_1_1_d_x_endpoint_a230f7841ff2da5b2da3d6d72bd5fb8a0}{DXEndpoint\+::create(\+Role)} factory method. Endpoint role is represented by \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2d}{DXEndpoint\+::Role} enumeration.

Endpoint role defines the behavior of its \doxylink{structdxfcpp_1_1_d_x_endpoint_a26dc7548a792c0e3eb0b47a90ff7771c}{connect} method\+:


\begin{DoxyItemize}
\item \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2dae352a08d4fba713e5ae1e538bf27258c}{FEED} connects to the remote data feed provider and is optimized for real-\/time or delayed data processing ({\bfseries{this is a default role}}). \doxylink{structdxfcpp_1_1_d_x_endpoint_aadf6d4f1eeb73b91c4f7f1deca4bb61f}{DXEndpoint\+::get\+Feed()} method returns a feed object that subscribes to this remote data feed provider and receives events from it. When event processing threads cannot keep up (don\textquotesingle{}t have enough CPU time), data is dynamically conflated to minimize latency between received events and their processing time. For example\+:

\doxyparagraph*{Endpoint state}

Each endpoint has a state that can be retrieved with \doxylink{structdxfcpp_1_1_d_x_endpoint_aaac4bccb8c818c9a3edfb59d98fe8b7c}{DXEndpoint\+::get\+State()} method. When endpoint is created with any role and default address is not specified in default properties, then it is not connected to any remote endpoint. Its state is \doxylink{structdxfcpp_1_1_d_x_endpoint_af82eb2c48281dedb4d2a1994f07a22c6ad91be9b5ceb9fe0af02b9b02413eccf8}{NOT\+\_\+\+CONNECTED}.

\doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2dae352a08d4fba713e5ae1e538bf27258c}{Feed} and \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2da02c25714ac6e2973dfb530a7520c80ae}{publisher} endpoints can connect to remote endpoints of the opposite role. Connection is initiated by \doxylink{structdxfcpp_1_1_d_x_endpoint_a26dc7548a792c0e3eb0b47a90ff7771c}{connect} method. The endpoint state becomes \doxylink{structdxfcpp_1_1_d_x_endpoint_af82eb2c48281dedb4d2a1994f07a22c6a9a14f95e151eec641316e7c784ce832d}{CONNECTING}.

When the actual connection to the remote endpoint is established, the endpoint state becomes \doxylink{structdxfcpp_1_1_d_x_endpoint_af82eb2c48281dedb4d2a1994f07a22c6aa5afd6edd5336d91316964e493936858}{CONNECTED}.

Network connections can temporarily break and return endpoint back into \doxylink{structdxfcpp_1_1_d_x_endpoint_af82eb2c48281dedb4d2a1994f07a22c6a9a14f95e151eec641316e7c784ce832d}{CONNECTING} state. File connections can be completed and return endpoint into \doxylink{structdxfcpp_1_1_d_x_endpoint_af82eb2c48281dedb4d2a1994f07a22c6ad91be9b5ceb9fe0af02b9b02413eccf8}{NOT\+\_\+\+CONNECTED} state.

Connection to the remote endpoint can be terminated with \doxylink{structdxfcpp_1_1_d_x_endpoint_a3cd3eb1eef171e3cc16e8a811e7c276e}{DXEndpoint\+::disconnect()} method. The endpoint state becomes \doxylink{structdxfcpp_1_1_d_x_endpoint_af82eb2c48281dedb4d2a1994f07a22c6ad91be9b5ceb9fe0af02b9b02413eccf8}{NOT\+\_\+\+CONNECTED}.

Endpoint can be closed with \doxylink{structdxfcpp_1_1_d_x_endpoint_a06b70688f7aa279e0a905099cb7cb4c9}{DXEndpoint\+::close()} method. The endpoint state becomes \doxylink{structdxfcpp_1_1_d_x_endpoint_af82eb2c48281dedb4d2a1994f07a22c6a110ccf2f5d2ff4eda1fd1a494293467d}{CLOSED}. This is a final state. All connection are terminated and all internal resources that are held by this endpoint are freed. No further connections can be initiated.

\doxyparagraph*{Event times}

The \doxylink{structdxfcpp_1_1_event_type_a7490dfa82916838dfdbea88c65157d3d}{Event\+Type\+::get\+Event\+Time()} on received events is available only when the endpoint is created with \doxylink{structdxfcpp_1_1_d_x_endpoint_ade95bc0b10e58f612fd4355aa915ade9}{DXEndpoint\+::\+DXENDPOINT\+\_\+\+EVENT\+\_\+\+TIME\+\_\+\+PROPERTY} property and the data source has embedded event times. This is typically true only for data events that are read from historical tape files (see above) and from On\+Demand\+Service. Events that are coming from a network connections do not have an embedded event time information and event time is not available for them anyway.

\doxyparagraph*{\label{structdxfcpp_1_1_d_x_endpoint_defaultPropertiesSection}%
\Hypertarget{structdxfcpp_1_1_d_x_endpoint_defaultPropertiesSection}%
Default properties}

Default properties are loaded from "{}dxfeed.\+properties"{} or "{}dxpublisher.\+properties"{} file depending on the \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2d}{role} of created endpoint. "{}dxfeed.\+properties"{} is used for \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2dae352a08d4fba713e5ae1e538bf27258c}{FEED} and \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2da4e0e3dbd8f06b71f54c84c97fea30e22}{ON\+\_\+\+DEMAND\+\_\+\+FEED}, "{}dxpublisher.\+properties"{} is used for \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2da02c25714ac6e2973dfb530a7520c80ae}{PUBLISHER}\}. \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2da08e8da2638e7a4a4dfb3d29ab5afb751}{STREAM\+\_\+\+FEED} and \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2da16722a7e83ac68f9bc32cae5242046f5}{LOCAL\+\_\+\+HUB} do not support properties file.

The location of this file can be specified using \doxylink{classdxfcpp_1_1_d_x_endpoint_1_1_builder_a06466a9c8a88df62c32c20e17788f4a4}{with\+Property}(\doxylink{structdxfcpp_1_1_d_x_endpoint_ac491fee2f147e882e63dc61fcf9e2d3b}{DXFEED\+\_\+\+PROPERTIES\+\_\+\+PROPERTY}, path) or \doxylink{classdxfcpp_1_1_d_x_endpoint_1_1_builder_a06466a9c8a88df62c32c20e17788f4a4}{with\+Property}(\doxylink{structdxfcpp_1_1_d_x_endpoint_a200a1641fd464f782b6715fc7d1c1d69}{DXPUBLISHER\+\_\+\+PROPERTIES\+\_\+\+PROPERTY}, path) correspondingly. When the location of this file is not explicitly specified using \doxylink{classdxfcpp_1_1_d_x_endpoint_1_1_builder_a06466a9c8a88df62c32c20e17788f4a4}{with\+Property} method, then the file path is taken from a system property with the corresponding name.

When the path to the above properties file is not provided, then a resource named "{}dxfeed.\+properties"{} or "{}dxpublisher.\+properties"{} is loaded from classpath. When classpath is set to "{}."{} (current directory), it means that the corresponding file can be placed into the current directory with any need to specify additional properties.

Defaults for individual properties can be also provided using system properties when they are not specified in the configuration file. \doxylink{structdxfcpp_1_1_system}{System} properties override configuration loaded from classpath resource, but don\textquotesingle{}t override configuration from the user-\/specified configuration file.

The \doxylink{structdxfcpp_1_1_d_x_endpoint_afe2a64464bb98673d09087043e5b2ff8}{DXEndpoint\+::\+NAME\+\_\+\+PROPERTY} is the exception from the above rule. It is never loaded from system properties. It can be only specified in configuration file or programmatically. There is a convenience \doxylink{classdxfcpp_1_1_d_x_endpoint_1_1_builder_ab4eba04d0142dcd63d24fe7a1d3cae5a}{Builder.with\+Name} method for it. It is recommended to assign short and meaningful endpoint names when multiple endpoints are used in the same process (one Graal\+VM Isolate for now). The name of the endpoint shall describe its role in the particular application.

Note, that individual properties that are programmatically set using \doxylink{classdxfcpp_1_1_d_x_endpoint_1_1_builder_a06466a9c8a88df62c32c20e17788f4a4}{with\+Property} method always take precedence.

\doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2dae352a08d4fba713e5ae1e538bf27258c}{FEED} and \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2da02c25714ac6e2973dfb530a7520c80ae}{PUBLISHER} automatically establish connection on creation when the corresponding \doxylink{structdxfcpp_1_1_d_x_endpoint_a6ad96851a0a4d3bd1453bdb2bc3a8c00}{DXEndpoint\+::\+DXFEED\+\_\+\+ADDRESS\+\_\+\+PROPERTY} or \doxylink{structdxfcpp_1_1_d_x_endpoint_a7494f59e150133d36b87c0184c2179eb}{DXEndpoint\+::\+DXPUBLISHER\+\_\+\+ADDRESS\+\_\+\+PROPERTY} is specified.

\doxyparagraph*{Permanent subscription}

Endpoint properties can define permanent subscription for specific sets of symbols and event types in the data feed, so that \doxylink{structdxfcpp_1_1_d_x_feed}{DXFeed} methods like get\+Last\+Event\+If\+Subscribed, get\+Indexed\+Events\+If\+Subscribed, and get\+Time\+Series\+If\+Subscribed can be used without a need to create a separate \doxylink{classdxfcpp_1_1_d_x_feed_subscription}{DXFeed\+Subscription} object. Please, contact dx\+Feed support for details on the required configuration.

This class is thread-\/safe and can be used concurrently from multiple threads without external synchronization.

Terminates all established network connections and initiates connecting again with the same address. 

The effect of the method is alike to invoking \doxylink{structdxfcpp_1_1_d_x_endpoint_a3cd3eb1eef171e3cc16e8a811e7c276e}{disconnect()} and \doxylink{structdxfcpp_1_1_d_x_endpoint_a26dc7548a792c0e3eb0b47a90ff7771c}{connect(const std\+::string\&)} with the current address, but internal resources used for connections may be reused by implementation. TCP connections with multiple target addresses will try switch to an alternative address, configured reconnect timeouts will apply.

{\bfseries{Note\+:}} The method will not connect endpoint that was not initially connected with \doxylink{structdxfcpp_1_1_d_x_endpoint_a26dc7548a792c0e3eb0b47a90ff7771c}{connect(const std\+::string\&)} method or was disconnected with \doxylink{structdxfcpp_1_1_d_x_endpoint_a3cd3eb1eef171e3cc16e8a811e7c276e}{disconnect()} method.
