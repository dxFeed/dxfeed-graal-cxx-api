\doxysubsubsection{Detailed Description}
\mbox{\hyperlink{classSeries}{Series}} event is a snapshot of computed values that are available for all option series for a given underlying symbol based on the option prices on the market. 

It represents the most recent information that is available about the corresponding values on the market at any given moment of time.

\mbox{\hyperlink{classSeries}{Series}} is an \mbox{\hyperlink{structIndexedEvent}{Indexed\+Event}} with multiple instances of event available for each underlying symbol. Each series event instance corresponds to an Option\+Series of the corresponding underlying. The correspondence between a series event instance and an Option\+Series is established via \mbox{\hyperlink{classSeries_ad9d84082306a664c5c1d7f0ca8b7fc6e}{expiration}} property. If case where there are multiple series at the same expiration day id, then series events are are ordered by their \mbox{\hyperlink{classSeries_a7813d76291d673ec40f1dbc4e91a957a}{index}} in the same order as the corresponding Option\+Series are ordered by their attributes.

\doxysubparagraph*{\label{_eventFlagsSection}%
Event flags, transactions and snapshots}

\mbox{\hyperlink{classSeries}{Series}} data source provides a consistent view of the set of known series. The corresponding information is carried in event\+Flags property. The logic behind this property is detailed in \mbox{\hyperlink{structIndexedEvent}{Indexed\+Event}} class documentation. Multiple event sources for the same symbol are not supported for series, thus source property is always \mbox{\hyperlink{classIndexedEventSource_ab359b82d74ee151707106b66eec0a4e3}{DEFAULT}}.

Indexed\+Event\+Model class handles all the snapshot and transaction logic and conveniently represents a list current of events.

\doxysubparagraph*{Implementation details}

This event is implemented on top of QDS records {\ttfamily \mbox{\hyperlink{classSeries}{Series}}}.
