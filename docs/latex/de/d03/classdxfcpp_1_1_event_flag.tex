\doxysubsubsection{Detailed Description}
\doxyparagraph*{\label{classdxfcpp_1_1_underlying_eventFlagsSection}%
\Hypertarget{classdxfcpp_1_1_underlying_eventFlagsSection}%
Event flags, transactions and snapshots}

Some indexed event sources provide a consistent view of a set of events for a given symbol. Their updates may incorporate multiple changes that have to be processed at the same time. The corresponding information is carried in \doxylink{structdxfcpp_1_1_indexed_event_a45ece86bfc828bf8a40ef1def0ec625e}{event\+Flags} property.

The value of \doxylink{structdxfcpp_1_1_indexed_event_a45ece86bfc828bf8a40ef1def0ec625e}{event\+Flags} property has several significant bits that are packed into an integer in the following way\+:


\begin{DoxyPre}{\ttfamily 
   31..7    6    5    4    3    2    1    0
+-\/-\/-\/-\/-\/-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+
|         | SM |    | SS | SE | SB | RE | TX |
+-\/-\/-\/-\/-\/-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+
}\end{DoxyPre}


Each source updates its transactional state using these bits separately. The state of each source has to be tracked separately in a map for each source. However, event \doxylink{structdxfcpp_1_1_indexed_event_ad5a71500687f8cebb2a15964144638dc}{index} is unique across the sources. This is achieved by allocating an event-\/specific number of most significant bits of \doxylink{structdxfcpp_1_1_indexed_event_ad5a71500687f8cebb2a15964144638dc}{index} for use as a \doxylink{structdxfcpp_1_1_indexed_event_af9f56273cba451eb0653e8236f6b2748}{source} \doxylink{classdxfcpp_1_1_indexed_event_source_a251e43b2d80bda9d117e8dd1c3aa5413}{id}.

{\ttfamily TX} (bit 0) --- \doxylink{classdxfcpp_1_1_event_flag_ae57282251576b5b5a09d93c0b588a3d7}{TX\+\_\+\+PENDING} is an indicator of pending transactional update. It can be retrieved from {\ttfamily event\+Flags} with the following piece of code\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{bool}\ txPending\ =\ (\textcolor{keyword}{event}-\/>\ getEventFlags()\ \&\ \mbox{\hyperlink{structdxfcpp_1_1_indexed_event_aa0fa82f8632275341f08649124aa89e3}{IndexedEvent::TX\_PENDING}})\ !=\ 0;}

\end{DoxyCode}


When {\ttfamily tx\+Pending} is {\ttfamily true} it means, that an ongoing transaction update that spans multiple events is in process. All events with {\ttfamily tx\+Pending} {\ttfamily true} shall be put into a separate {\itshape pending list} for each source id and should be processed later when an event for this source id with {\ttfamily tx\+Pending} {\ttfamily false} comes.

{\ttfamily RE} (bit 1) --- \doxylink{classdxfcpp_1_1_event_flag_ae244786e6fb0f520858ac29c08855708}{REMOVE\+\_\+\+EVENT} is used to indicate that that the event with the corresponding index has to be removed.


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{bool}\ removeEvent\ =\ (\textcolor{keyword}{event}-\/>\ getEventFlags()\ \&\ \mbox{\hyperlink{structdxfcpp_1_1_indexed_event_ae3cd8ed3be94e645913ccd23bfc07edd}{IndexedEvent::REMOVE\_EVENT}})\ !=\ 0;}

\end{DoxyCode}


{\ttfamily SB} (bit 2) --- \doxylink{classdxfcpp_1_1_event_flag_a440532930f74605794158bb59a161c59}{SNAPSHOT\+\_\+\+BEGIN} is used to indicate when the loading of a snapshot starts.


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{bool}\ snapshotBegin\ =\ (\textcolor{keyword}{event}-\/>\ getEventFlags()\ \&\ \mbox{\hyperlink{structdxfcpp_1_1_indexed_event_aacdbdadf8af938b9edbe9a59830aa0b5}{IndexedEvent::SNAPSHOT\_BEGIN}})\ !=\ 0;}

\end{DoxyCode}


Snapshot load starts on new subscription and the first indexed event that arrives for each non-\/zero source id on new subscription may have {\ttfamily snapshot\+Begin} set to {\ttfamily true}. It means, that an ongoing snapshot consisting of multiple events is incoming. All events for this source id shall be put into a separate {\itshape pending list} for each source id.

{\ttfamily SE} (bit 3) --- \doxylink{classdxfcpp_1_1_event_flag_acd462d106e5f20a5fe315bca6cdc31e0}{SNAPSHOT\+\_\+\+END} or {\ttfamily SS} (bit 4) --- \doxylink{classdxfcpp_1_1_event_flag_a138fa2278b8ca4800c8c0b7a402ddac0}{SNAPSHOT\+\_\+\+SNIP} are used to indicate the end of a snapshot.


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{bool}\ snapshotEnd\ =\ (\textcolor{keyword}{event}-\/>getEventFlags()\ \&\ \mbox{\hyperlink{structdxfcpp_1_1_indexed_event_a8140c207eb5efb3f656842509700366f}{IndexedEvent::SNAPSHOT\_END}})\ !=\ 0;}
\DoxyCodeLine{\textcolor{keywordtype}{bool}\ snapshotSnip\ =\ (\textcolor{keyword}{event}-\/>getEventFlags()\ \&\ \mbox{\hyperlink{structdxfcpp_1_1_indexed_event_a0615ae7acfc0032e03ada68c26433aa5}{IndexedEvent::SNAPSHOT\_SNIP}})\ !=\ 0;}

\end{DoxyCode}


The last event of a snapshot is marked with either {\ttfamily snapshot\+End} or {\ttfamily snapshot\+Snip}. At this time, all events from a {\itshape pending list} for the corresponding source can be processed, unless {\ttfamily tx\+Pending} is also set to {\ttfamily true}. In the later case, the processing shall be further delayed due to ongoing transaction.

The difference between {\ttfamily snapshot\+End} and {\ttfamily snapshot\+Snip} is the following. {\ttfamily snapshot\+End} indicates that the data source had sent all the data pertaining to the subscription for the corresponding indexed event, while {\ttfamily snapshot\+Snip} indicates that some limit on the amount of data was reached and while there still might be more data available, it will not be provided.

{\ttfamily SM} (bit 6) --- \doxylink{classdxfcpp_1_1_event_flag_a0688aae67c8839a36ce85e634503a8f5}{SNAPSHOT\+\_\+\+MODE} is used to instruct dx\+Feed to use snapshot mode. It is intended to be used only for publishing to activate (if not yet activated) snapshot mode. The difference from \doxylink{classdxfcpp_1_1_event_flag_a440532930f74605794158bb59a161c59}{SNAPSHOT\+\_\+\+BEGIN} flag is that \doxylink{classdxfcpp_1_1_event_flag_a0688aae67c8839a36ce85e634503a8f5}{SNAPSHOT\+\_\+\+MODE} only switches on snapshot mode without starting snapshot synchronization protocol.

When a snapshot is empty or consists of a single event, then the event can have both {\ttfamily snapshot\+Begin} and {\ttfamily snapshot\+End} or {\ttfamily snapshot\+Snip} flags. In case of an empty snapshot, {\ttfamily remove\+Event} on this event is also set to {\ttfamily true}.
