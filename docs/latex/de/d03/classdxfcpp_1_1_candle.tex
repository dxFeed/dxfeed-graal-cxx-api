\doxysubsubsection{Detailed Description}
\doxylink{classdxfcpp_1_1_candle}{Candle} event with open, high, low, close prices and other information for a specific period. 

Candles are build with a specified \doxylink{structdxfcpp_1_1_candle_period}{Candle\+Period} using a specified \doxylink{structdxfcpp_1_1_candle_price}{Candle\+Price} type with a data taken from the specified \doxylink{structdxfcpp_1_1_candle_exchange}{Candle\+Exchange} from the specified \doxylink{structdxfcpp_1_1_candle_session}{Candle\+Session} with further details of aggregation provided by \doxylink{structdxfcpp_1_1_candle_alignment}{Candle\+Alignment}.

Event symbol of the candle is represented with \doxylink{structdxfcpp_1_1_candle_symbol}{Candle\+Symbol} class. Since the \`{}\+Candle is a time-\/series event, it is typically subscribed to using DXFeed\+Time\+Series\+Subscription class that handles the necessarily wrapping of the symbol into \doxylink{classdxfcpp_1_1_time_series_subscription_symbol}{Time\+Series\+Subscription\+Symbol} to specify a subscription time range.

\doxyparagraph*{\label{classdxfcpp_1_1_underlying_eventFlagsSection}%
\Hypertarget{classdxfcpp_1_1_underlying_eventFlagsSection}%
Event flags, transactions and snapshots}

Some candle sources provide a consistent view of the set of known candles. The corresponding information is carried in \doxylink{classdxfcpp_1_1_candle_a7ba850252e025efbd427207c0bfc7de4}{event\+Flags} property. The logic behind this property is detailed in \doxylink{structdxfcpp_1_1_indexed_event}{Indexed\+Event} class documentation. Multiple event sources for the same symbol are not supported for candles, thus \doxylink{structdxfcpp_1_1_indexed_event_af9f56273cba451eb0653e8236f6b2748}{source} property is always \doxylink{classdxfcpp_1_1_indexed_event_source_ab7bc1a5037c8e18b2adeb45237cea5eb}{DEFAULT}.

Time\+Series\+Event\+Model class handles all the snapshot and transaction logic and conveniently represents a list current of time-\/series events order by their \doxylink{classdxfcpp_1_1_candle_a7439fe39a66e26ec0d4ae5aacd632f3a}{time}. It relies on the code of Abstract\+Indexed\+Event\+Model to handle this logic. Use the source code of Abstract\+Indexed\+Event\+Model for clarification on transactions and snapshot logic.

\doxyparagraph*{Publishing Candles}

Publishing of candle events follows the general rules explained in \doxylink{structdxfcpp_1_1_time_series_event}{Time\+Series\+Event} class documentation.

\doxyparagraph*{Implementation details}

This event is implemented on top of QDS record {\ttfamily Trade\+History} for tick candles with \doxylink{structdxfcpp_1_1_candle_period_a7f4f822ea54c646b667db887c18e0768}{Candle\+Period\+::\+TICK}, records {\ttfamily \doxylink{classdxfcpp_1_1_trade}{Trade}.\texorpdfstring{$<$}{<}period\texorpdfstring{$>$}{>}} for a certain set of popular periods, and QDS record {\ttfamily \doxylink{classdxfcpp_1_1_candle}{Candle}} for arbitrary custom periods, with a set of {\ttfamily \doxylink{classdxfcpp_1_1_candle}{Candle}\{\texorpdfstring{$<$}{<}attributes\texorpdfstring{$>$}{>}\}} records for a popular combinations of custom candle symbol attributes like \doxylink{structdxfcpp_1_1_candle_price}{Candle\+Price} for an efficient support of bid-\/ask charting.
