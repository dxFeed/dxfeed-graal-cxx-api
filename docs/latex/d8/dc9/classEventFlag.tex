\doxysubsubsection{Detailed Description}
\doxysubparagraph*{\label{_eventFlagsSection}%
Event flags, transactions and snapshots}

Some indexed event sources provide a consistent view of a set of events for a given symbol. Their updates may incorporate multiple changes that have to be processed at the same time. The corresponding information is carried in \mbox{\hyperlink{structIndexedEvent_a5bf54899ccd5dd4bd162ae64fbcc695b}{event\+Flags}} property.

The value of \mbox{\hyperlink{structIndexedEvent_a5bf54899ccd5dd4bd162ae64fbcc695b}{event\+Flags}} property has several significant bits that are packed into an integer in the following way\+:


\begin{DoxyPre}{\ttfamily 
   31..7    6    5    4    3    2    1    0
+-\/-\/-\/-\/-\/-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+
|         | SM |    | SS | SE | SB | RE | TX |
+-\/-\/-\/-\/-\/-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+-\/-\/-\/-\/+
}\end{DoxyPre}


Each source updates its transactional state using these bits separately. The state of each source has to be tracked separately in a map for each source. However, event \mbox{\hyperlink{structIndexedEvent_a4310a26785d5142efba9d7c0094125bc}{index}} is unique across the sources. This is achieved by allocating an event-\/specific number of most significant bits of \mbox{\hyperlink{structIndexedEvent_a4310a26785d5142efba9d7c0094125bc}{index}} for use as a \mbox{\hyperlink{structIndexedEvent_a54d06fa64c5f66ed78b328f4c36a633b}{source}} \mbox{\hyperlink{classIndexedEventSource_acd2b67cd96f688d1255b45783e9a1df9}{id}}.

{\ttfamily TX} (bit 0) --- \+::\+TX\+\_\+\+PENDING is an indicator of pending transactional update. It can be retrieved from {\ttfamily event\+Flags} with the following piece of code\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{bool} txPending = (\textcolor{keyword}{event}-\/> getEventFlags() \& \mbox{\hyperlink{structIndexedEvent_a11615cb90f791068cfa9edaa8b0056ae}{IndexedEvent::TX\_PENDING}}) != 0;}

\end{DoxyCode}


When {\ttfamily tx\+Pending} is {\ttfamily true} it means, that an ongoing transaction update that spans multiple events is in process. All events with {\ttfamily tx\+Pending} {\ttfamily true} shall be put into a separate {\itshape pending list} for each source id and should be processed later when an event for this source id with {\ttfamily tx\+Pending} {\ttfamily false} comes.

{\ttfamily RE} (bit 1) --- \+::\+REMOVE\+\_\+\+EVENT is used to indicate that that the event with the corresponding index has to be removed.


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{bool} removeEvent = (\textcolor{keyword}{event}-\/> getEventFlags() \& \mbox{\hyperlink{structIndexedEvent_a98e078c04445c983d19db700df469b58}{IndexedEvent::REMOVE\_EVENT}}) != 0;}

\end{DoxyCode}


{\ttfamily SB} (bit 2) --- \+::\+SNAPSHOT\+\_\+\+BEGIN is used to indicate when the loading of a snapshot starts.


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{bool} snapshotBegin = (\textcolor{keyword}{event}-\/> getEventFlags() \& \mbox{\hyperlink{structIndexedEvent_ac87eb588da470259ee75b490dbcaabf7}{IndexedEvent::SNAPSHOT\_BEGIN}}) != 0;}

\end{DoxyCode}


Snapshot load starts on new subscription and the first indexed event that arrives for each non-\/zero source id on new subscription may have {\ttfamily snapshot\+Begin} set to {\ttfamily true}. It means, that an ongoing snapshot consisting of multiple events is incoming. All events for this source id shall be put into a separate {\itshape pending list} for each source id.

{\ttfamily SE} (bit 3) --- \+::\+SNAPSHOT\+\_\+\+END or {\ttfamily SS} (bit 4) --- \+::\+SNAPSHOT\+\_\+\+SNIP are used to indicate the end of a snapshot.


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{bool} snapshotEnd = (\textcolor{keyword}{event}-\/>getEventFlags() \& \mbox{\hyperlink{structIndexedEvent_a4604235ec9a455bea7c80d1b71546bf9}{IndexedEvent::SNAPSHOT\_END}}) != 0;}
\DoxyCodeLine{\textcolor{keywordtype}{bool} snapshotSnip = (\textcolor{keyword}{event}-\/>getEventFlags() \& \mbox{\hyperlink{structIndexedEvent_a8824670805f7487194b7070fe06afe47}{IndexedEvent::SNAPSHOT\_SNIP}}) != 0;}

\end{DoxyCode}


The last event of a snapshot is marked with either {\ttfamily snapshot\+End} or {\ttfamily snapshot\+Snip}. At this time, all events from a {\itshape pending list} for the corresponding source can be processed, unless {\ttfamily tx\+Pending} is also set to {\ttfamily true}. In the later case, the processing shall be further delayed due to ongoing transaction.

The difference between {\ttfamily snapshot\+End} and {\ttfamily snapshot\+Snip} is the following. {\ttfamily snapshot\+End} indicates that the data source had sent all the data pertaining to the subscription for the corresponding indexed event, while {\ttfamily snapshot\+Snip} indicates that some limit on the amount of data was reached and while there still might be more data available, it will not be provided.

{\ttfamily SM} (bit 6) --- \+::\+SNAPSHOT\+\_\+\+MODE is used to instruct dx\+Feed to use snapshot mode. It is intended to be used only for publishing to activate (if not yet activated) snapshot mode. The difference from \+::\+SNAPSHOT\+\_\+\+BEGIN flag is that \+::\+SNAPSHOT\+\_\+\+MODE only switches on snapshot mode without starting snapshot synchronization protocol.

When a snapshot is empty or consists of a single event, then the event can have both {\ttfamily snapshot\+Begin} and {\ttfamily snapshot\+End} or {\ttfamily snapshot\+Snip} flags. In case of an empty snapshot, {\ttfamily remove\+Event} on this event is also set to {\ttfamily true}.
