\doxysubsubsection{Detailed Description}
Represents an extension of \mbox{\hyperlink{classOrder}{Order}} for the symbols traded on the OTC Markets. 

It includes the OTC Markets specific quote data, such as \mbox{\hyperlink{classOtcMarketsOrder_a85155c31cda80ccd911f84389efcd666}{Quote Access Payment (QAP) value}}, \mbox{\hyperlink{classOtcMarketsOrder_aee3c7b083cf41b41457bde00177108b7}{Quote Price Type}}, \mbox{\hyperlink{classOtcMarketsOrder_ac37e7a0ebb4051f965125e608640165d}{Open/\+Closed status}}, and \mbox{\hyperlink{classOtcMarketsOrder_abdc79206ae54aaa371a6ee46593cabc9}{Unsolicited quote indicator}}. The original event (OTC \mbox{\hyperlink{classQuote}{Quote}}) published by the exchange is presented in the form of two separate OTCMarkets\+Order events (bid and ask side). Note that the description of the fields (e.\+g. QAP) uses the original \char`\"{}quote\char`\"{} term. 

For more information about original fields, QAP, \mbox{\hyperlink{classQuote}{Quote}} Flags and Extended \mbox{\hyperlink{classQuote}{Quote}} Flags, see \href{https://downloads.dxfeed.com/specifications/OTC_Markets_Multicast_Data_Feeds.pdf}{\texttt{ OTC Markets Multicast Data Feed}}. 

For more information about display requirements, see \href{https://downloads.dxfeed.com/specifications/OTC_Markets_Data_Display_Requirements.pdf}{\texttt{ OTC Markets Display Requirements}}.

Like regular orders, OTC Markets order events arrive from multiple sources for the same market symbol and are distinguished by their index. It is unique across all the sources of depth information for the symbol. The event with size either {\ttfamily 0} or {\ttfamily NaN} is a signal to remove previously received order for the corresponding index. The method \+::has\+Size() is a convenient method to test for size presence.

\doxysubparagraph*{\label{_eventFlagsSection}%
Event flags, transactions and snapshots}

OTC Markets order event sources provide a consistent view of the order book. Their updates may incorporate multiple changes to individual orders that have to be processed at the same time. The corresponding information is carried in event\+Flags property.

See the \char`\"{}\+Event Flags\char`\"{} section of \mbox{\hyperlink{classOrderBase}{Order\+Base}} class documentation for details.

\doxysubparagraph*{Publishing order books}

When publishing an order event with \mbox{\hyperlink{structDXPublisher_ae6e8cb6b370ea60f1ad5cb80cea2df30}{DXPublisher\+::publish\+Events()}} method, least significant 32 bits of order index must be in a range of from 0 to {\ttfamily std\+::numeric\+\_\+limits$<$std\+::int32\+\_\+t$>$\+::max()} inclusive. Use \+::set\+Source() method after \+::set\+Index() to properly include source identifier into the index.

A snapshot has to be published in the {\itshape descending} order of index, starting with an event with the largest index and marking it with \mbox{\hyperlink{classEventFlag_ab1246c3963485123672b05d5b9dbdfec}{Event\+Flag\+::\+SNAPSHOT\+\_\+\+BEGIN}} bit in event\+Flags, and finishing the snapshot with an event that has zero 32 least significant bits of index. \mbox{\hyperlink{classEventFlag_adc2ec8ed07a2a2efaf2389fe4f7459c6}{Event\+Flag\+::\+SNAPSHOT\+\_\+\+END}} bit in event\+Flags is optional during publishing. It will be properly set on receiving end anyway.

\doxysubparagraph*{Limitations}

This event type cannot be used with DXFeed\+::get\+Last\+Event() method.

Full \mbox{\hyperlink{classOrder}{Order}} Book Support}

\begin{DoxyItemize}
\item action -\/ event business meaning (see \mbox{\hyperlink{structOrderAction}{Order\+Action}} for more details) 
\item action\+Time -\/ time of the last action 
\item order\+Id -\/ ID of this OTC Markets order 
\item aux\+Order\+Id -\/ additional ID for this OTC Markets order 
\item trade\+Id -\/ trade (order execution) ID 
\item trade\+Price -\/ price of the trade 
\item trade\+Size -\/ size of the trade 
\end{DoxyItemize}
