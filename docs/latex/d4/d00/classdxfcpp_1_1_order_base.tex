\doxysubsubsection{Detailed Description}
Base class for common fields of \doxylink{classdxfcpp_1_1_order}{Order}, \doxylink{classdxfcpp_1_1_analytic_order}{Analytic\+Order} and \doxylink{classdxfcpp_1_1_spread_order}{Spread\+Order} events. 

\doxylink{classdxfcpp_1_1_order}{Order} events represent a snapshot for a full available market depth for a symbol. The collection of order events of a symbol represents the most recent information that is available about orders on the market at any given moment of time.

\doxylink{classdxfcpp_1_1_order}{Order} event represents market depth for a {\bfseries{specific symbol}}.

\doxylink{classdxfcpp_1_1_analytic_order}{Analytic\+Order} event represents market depth for a {\bfseries{specific symbol}} extended with an analytic information, for example, whether particular order represent an iceberg or not.

\doxylink{classdxfcpp_1_1_spread_order}{Spread\+Order} event represents market depth for {\bfseries{all spreads on a given underlying symbol}}.

\doxylink{classdxfcpp_1_1_order}{Order} events arrive from multiple sources for the same market symbol and are distinguished by their \doxylink{classdxfcpp_1_1_order_base_ae40f5bf3aadb65c03f0d512569a23b74}{index}. Index is a unique per symbol identifier of the event. It is unique across all the sources of depth information for the symbol. The event with \doxylink{classdxfcpp_1_1_order_base_a3772a63a9d2a79b8cb9a687a2b2c0a7a}{size} either {\ttfamily 0} or {\ttfamily NaN} is a signal to remove previously received order for the corresponding index. The method \doxylink{classdxfcpp_1_1_order_base_aa7c935513d5717625af3563761703350}{has\+Size} is a convenient method to test for size presence.

\doxyparagraph*{\label{classdxfcpp_1_1_underlying_eventFlagsSection}%
\Hypertarget{classdxfcpp_1_1_underlying_eventFlagsSection}%
Event flags, transactions and snapshots}

Some order event sources provide a consistent view of the price-\/level or detailed order book. Their updates may incorporate multiple changes to price levels or to individual orders that have to be processed at the same time. The corresponding information is carried in \doxylink{classdxfcpp_1_1_order_base_a6d1da10a6ba9bb4fcc3466a5c0b07ac5}{event\+Flags} property. The logic behind this property is detailed in \doxylink{structdxfcpp_1_1_indexed_event}{Indexed\+Event} class documentation.

The event \doxylink{classdxfcpp_1_1_order_base_ab1943c5bee398eda008d0330dcc58a87}{source} identifier for an order is a part of the unique event \doxylink{classdxfcpp_1_1_order_base_ae40f5bf3aadb65c03f0d512569a23b74}{index}. It occupies highest bits of the \doxylink{classdxfcpp_1_1_order_base_ae40f5bf3aadb65c03f0d512569a23b74}{index} (index is not-\/negative). The lowest bits of \doxylink{classdxfcpp_1_1_order_base_ae40f5bf3aadb65c03f0d512569a23b74}{index} contain source-\/specific event index which is always zero in an event that is marked with \doxylink{classdxfcpp_1_1_event_flag_acd462d106e5f20a5fe315bca6cdc31e0}{Event\+Flag\+::\+SNAPSHOT\+\_\+\+END} bit in \doxylink{classdxfcpp_1_1_order_base_a6d1da10a6ba9bb4fcc3466a5c0b07ac5}{event\+Flags}.

Note that for an order with \doxylink{classdxfcpp_1_1_event_flag_ae244786e6fb0f520858ac29c08855708}{Event\+Flag\+::\+REMOVE\+\_\+\+EVENT} bit in \doxylink{classdxfcpp_1_1_order_base_a6d1da10a6ba9bb4fcc3466a5c0b07ac5}{event\+Flags} it is always the case that \doxylink{classdxfcpp_1_1_order_base_a3772a63a9d2a79b8cb9a687a2b2c0a7a}{size} is either {\ttfamily 0} or {\ttfamily NaN}, so no additional logic to process this bit is required for orders. Transactions and snapshots may include orders with \doxylink{classdxfcpp_1_1_order_base_a3772a63a9d2a79b8cb9a687a2b2c0a7a}{size} of {\ttfamily 0}. The filtering that distinguishes those events as removals of orders shall be performed after all transactions and snapshot processing.

Some aggregated feeds (like CME market depth) are mapped into two distinct source ids (one for buy orders and one for sell orders), but updates and transactions may span both. It is important to keep a separate track of transactional state for each source id, but, at the same time, when DXFeed\+Event\+Listener\+::events\+Received() method is called for a list of events, the order book shall be considered complete and consistent only when all events from the given list are processed.
