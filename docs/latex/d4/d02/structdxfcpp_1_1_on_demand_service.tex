\doxysubsubsection{Detailed Description}
Provides on-\/demand historical tick data replay controls. 

This class is used to seamlessly transition from ordinary real-\/time or delayed data feed to the replay of the tick-\/by-\/tick history of market data behaviour without any changes to the code that consumes and process these market events.

As single \doxylink{structdxfcpp_1_1_on_demand_service}{On\+Demand\+Service} instance is conceptually associated with each \doxylink{structdxfcpp_1_1_d_x_endpoint}{DXEndpoint} endpoint instance with the role of \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2dae352a08d4fba713e5ae1e538bf27258c}{FEED} or \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2da4e0e3dbd8f06b71f54c84c97fea30e22}{ON\+\_\+\+DEMAND\+\_\+\+FEED}. You can retrieve \doxylink{structdxfcpp_1_1_on_demand_service}{On\+Demand\+Service} instance that is associated with a given \doxylink{structdxfcpp_1_1_d_x_endpoint}{DXEndpoint} using \doxylink{structdxfcpp_1_1_on_demand_service_ae15499a709aa000ba8debd302eafa4ab}{get\+Instance(endpoint)} method.

For example, \doxylink{structdxfcpp_1_1_on_demand_service}{On\+Demand\+Service} instance for a default \doxylink{structdxfcpp_1_1_d_x_feed}{DXFeed} that is featured in the documentation of \doxylink{structdxfcpp_1_1_d_x_feed}{DXFeed} class and is retrieved via \doxylink{structdxfcpp_1_1_d_x_feed_ace4eaf9e79186f44e989d97622e0dcf0}{DXFeed\+::get\+Instance()} method can be acquired with the following code\+: 
\begin{DoxyPre}{\ttfamily 
\doxylink{structdxfcpp_1_1_d_x_endpoint}{auto} endpoint = \doxylink{structdxfcpp_1_1_d_x_endpoint_a6b190bf5c614f618a80ae359c6334cb6}{DXEndpoint::getInstance()};
\doxylink{structdxfcpp_1_1_on_demand_service}{auto} onDemand = \doxylink{structdxfcpp_1_1_on_demand_service}{OnDemandService}::\doxylink{structdxfcpp_1_1_on_demand_service_ae15499a709aa000ba8debd302eafa4ab}{getInstance}(endpoint);
}\end{DoxyPre}


This instance can be used for on-\/demand historical tick data replay only when endpoint is connected to the on-\/demand data provider and the appropriate credentials are provided. \doxylink{structdxfcpp_1_1_on_demand_service_a90d8fbd5848ea7ddf107649e39e634dd}{is\+Replay\+Supported} returns {\ttfamily true} when it is so.

On-\/demand replay is started with replay method that takes time as a parameter. \doxylink{structdxfcpp_1_1_d_x_feed}{DXFeed} is then disconnected from other data providers, pre-\/buffers historical tick data for replay, and starts replay of the data as if it was coming from the data feeds now. All the usual APIs that are part of \doxylink{structdxfcpp_1_1_d_x_feed}{DXFeed} like \doxylink{classdxfcpp_1_1_d_x_feed_subscription}{subscription} and various dx\+Feed models can be used normally as with the ordinary real-\/time or delayed data feed.

Replay speed can be changed on the fly with set\+Speed method or it can be set initially when starting replay with a two-\/argument version of replay method. \doxylink{structdxfcpp_1_1_on_demand_service_ad6320537ee56243d6706aa65860a232d}{pause} method is the same as set\+Speed(0).

\doxylink{structdxfcpp_1_1_on_demand_service_a0ab8f606931c2a69624b4edeaa15a11e}{stop\+And\+Resume} method stops data replay and resumes ordinary real-\/time or delayed data feed that was used before replay method was invoked. Endpoints with a role of \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2da4e0e3dbd8f06b71f54c84c97fea30e22}{ON\+\_\+\+DEMAND\+\_\+\+FEED} do not have an ordinary feed (they are essentially on-\/demand only) and thus \doxylink{structdxfcpp_1_1_on_demand_service_a0ab8f606931c2a69624b4edeaa15a11e}{stop\+And\+Resume} method works like \doxylink{structdxfcpp_1_1_on_demand_service_ac86fee581ed9953977a3bd7ea1dca402}{stop\+And\+Clear} for them.

\doxyparagraph*{State and change notifications}

On-\/demand historical tick data replay state can be queried with \doxylink{structdxfcpp_1_1_on_demand_service_a90d8fbd5848ea7ddf107649e39e634dd}{is\+Replay\+Supported}, \doxylink{structdxfcpp_1_1_on_demand_service_a60d12010746e25e6d5a49dac7269ec84}{is\+Replay}, \doxylink{structdxfcpp_1_1_on_demand_service_aea890fdc0f949d1beadc77eeaf9a720d}{get\+Speed}, and \doxylink{structdxfcpp_1_1_on_demand_service_a2d5e98f573a8fc2d7d5c5bce1714e82b}{get\+Time} methods.

\doxyparagraph*{Threads and locks}

This class is thread-\/safe and can be used concurrently from multiple threads without external synchronization.
