\doxysubsubsection{Detailed Description}
\doxylink{class_order}{Order} event is a snapshot for a full available market depth for a symbol. 

The collection of order events of a symbol represents the most recent information that is available about orders on the market at any given moment of time. \doxylink{class_order}{Order} events give information on several levels of details, called scopes -\/ see \doxylink{struct_scope}{Scope}. \doxylink{struct_scope}{Scope} of an order is available via \doxylink{class_order_base_ad8c08003e962e365a5b3266d3b5db801}{scope} property.

\doxylink{class_order}{Order} events arrive from multiple sources for the same market symbol and are distinguished by their \doxylink{class_order_base_a21bc36fb881577927015977bc5f71ec6}{index}. Index is a unique per symbol identifier of the event. It is unique across all the sources of depth information for the symbol. The event with \doxylink{class_order_base_a1a128e82af72d3fcd44a7f3eb8bcfd44}{size} either {\ttfamily 0} or {\ttfamily NaN} is a signal to remove previously received order for the corresponding index. The method \doxylink{class_order_base_afb5bea9f67fcdd11410562ab462b901d}{Order\+::has\+Size()} is a convenient method to test for size presence.

Events from finer-\/grained \doxylink{struct_scope}{Scope} of detail give more information and include events from coarse-\/grained \doxylink{struct_scope}{Scope} of detail. For a consistent representation of the market depth to the end-\/user, order events must be filtered to remove coarse-\/grained events that are superseded by the finer-\/grained events.

\doxysubsubsubsubsection*{\label{class_underlying_eventFlagsSection}%
\Hypertarget{class_underlying_eventFlagsSection}%
Event flags, transactions and snapshots}

Some order event sources provide a consistent view of the price-\/level or detailed order book. Their updates may incorporate multiple changes to price levels or to individual orders that have to be processed at the same time. The corresponding information is carried in \doxylink{class_order_base_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags} property.

See {\ttfamily Event Flags} section of \doxylink{class_order_base}{Order\+Base} class documentation for details.

The composite quotes with \doxylink{struct_scope_ad5b8601448fc3bf4b6406a5e26cd3dac}{Scope\+::\+COMPOSITE} and regional quotes with \doxylink{struct_scope_ad5ba143a0570faeb8c5c041a8ca93b8f}{Scope\+::\+REGIONAL} come individually from different venues and are not related to each other in any transactional way. The result of \doxylink{class_order_base_af5e6a4420f059a8e46e664dafebb6ea4}{Order\+::get\+Event\+Flags} method for them is always zero.

\doxysubsubsubsubsection*{Publishing order books}

When publishing an order event with \doxylink{struct_d_x_publisher_ae6e8cb6b370ea60f1ad5cb80cea2df30}{DXPublisher\+::publish\+Events()} method, least significant 32 bits of order \doxylink{class_order_base_a21bc36fb881577927015977bc5f71ec6}{index} must be in a range of from 0 to {\ttfamily std\+::numeric\+\_\+limits\texorpdfstring{$<$}{<}std\+::int32\+\_\+t\texorpdfstring{$>$}{>}::max()} inclusive. Use \doxylink{class_order_base_a3451d2eb5655cd3c902c78e3368ecfe3}{Order\+::set\+Source()} method after \doxylink{class_order_base_a1a7ce2fb19329a2f8450110cb58b9ac0}{Order\+::set\+Index()} to properly include source identifier into the index.

A snapshot has to be published in the {\itshape descending} order of \doxylink{class_order_base_a21bc36fb881577927015977bc5f71ec6}{index}, starting with an event with the largest index and marking it with \doxylink{class_event_flag_ab1246c3963485123672b05d5b9dbdfec}{Event\+Flag\+::\+SNAPSHOT\+\_\+\+BEGIN} bit in \doxylink{class_order_base_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags}, and finishing the snapshot with an event that has zero 32 least significant bits of index. \doxylink{class_event_flag_adc2ec8ed07a2a2efaf2389fe4f7459c6}{Event\+Flag\+::\+SNAPSHOT\+\_\+\+END} bit in \doxylink{class_order_base_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags} is optional during publishing. It will be properly set on receiving end anyway.

\doxysubsubsubsubsection*{Limitations}

This event type cannot be used with DXFeed\+::get\+Last\+Event() method.

\doxysubsubsubsubsection*{Implementation details}

This event is implemented on top of QDS record {\ttfamily \doxylink{class_quote}{Quote}} for composite quotes with \doxylink{struct_scope_ad5b8601448fc3bf4b6406a5e26cd3dac}{Scope\+::\+COMPOSITE}, record {\ttfamily \doxylink{class_quote}{Quote}\&X} for regional exchange best quotes with \doxylink{struct_scope_ad5ba143a0570faeb8c5c041a8ca93b8f}{Scope\+::\+REGIONAL}, record {\ttfamily Market\+Maker} for market-\/maker quotes and futures price level aggregates with \doxylink{struct_scope_a93aa5fc2d20e8f6a513e64cc5b906113}{Scope\+::\+AGGREGATE}, record {\ttfamily \doxylink{class_order}{Order}} for the most fine-\/grained \doxylink{struct_scope_a9d2f3400ab0b7c487262f11d31b8cc3e}{Scope\+::\+ORDER} with zero source id, and records {\ttfamily \doxylink{class_order}{Order}\#\texorpdfstring{$<$}{<}source-\/id\texorpdfstring{$>$}{>}} for specific source ids, where {\ttfamily \texorpdfstring{$<$}{<}source-\/id\texorpdfstring{$>$}{>}} is up to 4 ASCII characters with a mnemonic for the source like "{}\+NTV"{}.
