\doxysubsubsection{Detailed Description}
Represents time-\/series snapshots of some process that is evolving in time or actual events in some external system that have an associated time stamp and can be uniquely identified. 

For example, \doxylink{class_time_and_sale}{Time\+And\+Sale} events represent the actual sales that happen on a market exchange at specific time moments, while \doxylink{class_candle}{Candle} events represent snapshots of aggregate information about trading over a specific time period.

Time-\/series events can be used with DXFeed\+Time\+Series\+Subscription to receive a time-\/series history of past events. Time-\/series events are conflated based on unique \doxylink{struct_time_series_event_a8a4b57e2b6726274743c2b76689b693d}{index} for each symbol. Last indexed event for each symbol and index is always delivered to event listeners on subscription, but intermediate (next-\/to-\/last) events for each symbol+index pair are not queued anywhere, they are simply discarded as stale events.

Timestamp of an event is available via \doxylink{struct_time_series_event_a9d2ae41520f33140a6e823ae650e216a}{time} property with a millisecond precision. Some events may happen multiple times per millisecond. In this case they have the same \doxylink{struct_time_series_event_a9d2ae41520f33140a6e823ae650e216a}{time}, but different \doxylink{struct_time_series_event_a8a4b57e2b6726274743c2b76689b693d}{index}. An ordering of \doxylink{struct_time_series_event_a8a4b57e2b6726274743c2b76689b693d}{index} is the same as an ordering of \doxylink{struct_time_series_event_a9d2ae41520f33140a6e823ae650e216a}{time}. That is, an collection of time-\/series events that is ordered by \doxylink{struct_time_series_event_a8a4b57e2b6726274743c2b76689b693d}{index} is ascending order will be also ordered by \doxylink{struct_time_series_event_a9d2ae41520f33140a6e823ae650e216a}{time} in ascending order.

Time-\/series events are a more specific subtype of \doxylink{struct_indexed_event}{Indexed\+Event}. All general documentation and \href{IndexedEvent.html\#eventFlagsSection}{\texttt{ Event Flags}} section, in particular, applies to time-\/series events. However, the time-\/series events never come from multiple sources for the same symbol and their \doxylink{struct_time_series_event_a3c0a02028f2c42e8908f5bcdda02c0ff}{source} is always \doxylink{class_indexed_event_source_ab359b82d74ee151707106b66eec0a4e3}{DEFAULT}.

Unlike a general \doxylink{struct_indexed_event}{Indexed\+Event} that is subscribed to via \doxylink{class_d_x_feed_subscription}{DXFeed\+Subscription}\} using a plain symbol to receive all events for all indices, time-\/series events are typically subscribed to using \doxylink{class_time_series_subscription_symbol}{Time\+Series\+Subscription\+Symbol} class to specific time range of the subscription. There is a dedicated DXFeed\+Time\+Series\+Subscription class that is designed to simplify the task of subscribing for time-\/series events.

Time\+Series\+Event\+Model class handles all the snapshot and transaction logic and conveniently represents a list of current time-\/series events ordered by their \doxylink{struct_time_series_event_a9d2ae41520f33140a6e823ae650e216a}{time}. It relies on the code of Abstract\+Indexed\+Event\+Model to handle this logic. Use the source code of Abstract\+Indexed\+Event\+Model for clarification on transactions and snapshot logic.

Classes that implement this interface may also implement \doxylink{struct_lasting_event}{Lasting\+Event} interface, which makes it possible to use DXFeed\+::get\+Last\+Event() method to retrieve the last event for the corresponding symbol.

\doxysubsubsubsubsection*{Publishing time-\/series}

When publishing time-\/series event with \doxylink{struct_d_x_publisher_ae6e8cb6b370ea60f1ad5cb80cea2df30}{DXPublisher\+::publish\+Events()} method on incoming \doxylink{class_time_series_subscription_symbol}{Time\+Series\+Subscription\+Symbol} the snapshot of currently known events for the requested time range has to be published first.

A snapshot is published in the {\itshape descending} order of \doxylink{struct_time_series_event_a8a4b57e2b6726274743c2b76689b693d}{index} (which is the same as the descending order of \doxylink{struct_time_series_event_a9d2ae41520f33140a6e823ae650e216a}{time}), starting with an event with the largest index and marking it with \doxylink{struct_indexed_event_ac87eb588da470259ee75b490dbcaabf7}{Time\+Series\+Event\+::\+SNAPSHOT\+\_\+\+BEGIN} bit in \doxylink{struct_indexed_event_a5bf54899ccd5dd4bd162ae64fbcc695b}{event\+Flags}. All other event follow with default, zero \doxylink{struct_indexed_event_a5bf54899ccd5dd4bd162ae64fbcc695b}{event\+Flags}. If there is no actual event at the time that was specified in subscription \doxylink{class_time_series_subscription_symbol_a7ea676e77bdea236aa088925ce8a3c93}{from\+Time} property, then event with the corresponding time has to be created anyway and published. To distinguish it from the actual event, it has to be marked with \doxylink{struct_indexed_event_a98e078c04445c983d19db700df469b58}{Time\+Series\+Event\+::\+REMOVE\+\_\+\+EVENT} bit in \doxylink{struct_indexed_event_a5bf54899ccd5dd4bd162ae64fbcc695b}{event\+Flags}. \doxylink{struct_indexed_event_a4604235ec9a455bea7c80d1b71546bf9}{Time\+Series\+Event\+::\+SNAPSHOT\+\_\+\+END} bit in this event\textquotesingle{}s \doxylink{struct_indexed_event_a5bf54899ccd5dd4bd162ae64fbcc695b}{event\+Flags} s optional during publishing. It will be properly set on receiving end anyway. Note, that publishing any event with time that is below subscription \doxylink{class_time_series_subscription_symbol_a7ea676e77bdea236aa088925ce8a3c93}{from\+Time} also works as a legal indicator for the end of the snapshot.

Both \doxylink{class_time_and_sale}{Time\+And\+Sale} and \doxylink{class_candle}{Candle} time-\/series events define a sequence property that is mixed into an \doxylink{struct_time_series_event_a8a4b57e2b6726274743c2b76689b693d}{index} property. It provides a way to distinguish different events at the same time. For a snapshot end event the sequence has to be left at its default zero value. It means, that if there is an actual event to be published at subscription \doxylink{class_time_series_subscription_symbol_a7ea676e77bdea236aa088925ce8a3c93}{from\+Time} with non-\/zero sequence, then generation of an additional snapshot end event with the subscription \doxylink{class_time_series_subscription_symbol_a7ea676e77bdea236aa088925ce8a3c93}{from\+Time} and zero sequence is still required.

\doxysubsubsubsubsection*{Implementation notes}

The most common scheme for event indices is to set highest 32 bits of event index to event timestamp in seconds. The lowest 32 bits are then split as follows. Bits 22 to 31 encode milliseconds of time stamp, and bits 0 to 21 encode some kind of a sequence number.

Ultimately, the scheme for event indices is specific for each even type. The actual classes for specific event types perform the corresponding encoding.
