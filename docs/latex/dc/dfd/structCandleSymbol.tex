\doxysubsubsection{Detailed Description}
Symbol that should be used with \mbox{\hyperlink{classDXFeedSubscription}{DXFeed\+Subscription}} class to subscribe for \mbox{\hyperlink{classCandle}{Candle}} events. 

{\ttfamily \mbox{\hyperlink{classDXFeedSubscription}{DXFeed\+Subscription}}} also accepts a string representation of the candle symbol for subscription.

\doxysubparagraph*{String representation}

The string representation of the candle symbol consist of a \mbox{\hyperlink{structCandleSymbol_a875bc774f58a04211a33023255f4ba92}{base\+Symbol}} followed by an optional \textquotesingle{}\&\textquotesingle{} with an \mbox{\hyperlink{structCandleSymbol_a85465d5ca4e8b2b90492b3e3c19edf0a}{exchange}} code letter and followed by an optional list of comma-\/separated key=value pairs in curly braces\+:

`$<$base\+Symbol$>$ \mbox{[} '\&\textquotesingle{} $<$exchange$>$ \mbox{]} \mbox{[} \textquotesingle{}\{\textquotesingle{} $<$key1$>$=$<$value1$>$ \mbox{[} \textquotesingle{},\textquotesingle{} $<$key2$>$=$<$value2$>$ \mbox{]} ... \textquotesingle{}\}\textquotesingle{} \mbox{]}\`{}

Properties of the candle symbol correspond to the keys in the string representation in the following way\+:


\begin{DoxyItemize}
\item Empty key corresponds to \mbox{\hyperlink{structCandleSymbol_a5ab7429feb5e3f391c3a7d1af32cf2b5}{period}} --- aggregation period of this symbol. The period value is composed of an optional \mbox{\hyperlink{structCandlePeriod_a1e2e33ae5421ab04b022a50db2a4ad0a}{value}} which defaults to 1 when not specified, followed by a \mbox{\hyperlink{structCandlePeriod_ad3a01c476df66e8f88e0a92752c180a0}{type}} string which is defined by one of the \mbox{\hyperlink{structCandleType}{Candle\+Type}} values and can be abbreviated to first letters. For example, a daily candle of \char`\"{}\+IBM\char`\"{} base symbol can be specified as \char`\"{}\+IBM\{=d\}\char`\"{} and 15 minute candle on it as \char`\"{}\+IBM\{=15m\}\char`\"{}. The shortest possible abbreviation for \mbox{\hyperlink{structCandleType_a11b9390c342db5b8d2d8c17847c85880}{Candle\+Type\+::\+MONTH}} is \char`\"{}mo\char`\"{}, so the monthly candle can be specified as \char`\"{}\+IBM\{=mo\}\char`\"{}. When period is not specified, then the \mbox{\hyperlink{structCandlePeriod_a2934f6797a1c4aeb8ff49560370f0550}{Candle\+Period\+::\+TICK}} aggregation period is assumed as default. Note, that tick aggregation may not be available on the demo system which is limited to a subset of symbols and aggregation periods. 
\item \char`\"{}price\char`\"{} key corresponds to \mbox{\hyperlink{structCandleSymbol_a470e7d3b9a9298b8f0ce3c25a46f319d}{price}} --- price type attribute of this symbol. The \mbox{\hyperlink{structCandlePrice}{Candle\+Price}} defines possible values with \mbox{\hyperlink{structCandlePrice_a81644d93d41d6cd64bad82961637ca54}{Candle\+Price\+::\+LAST}} being default. For legacy backwards-\/compatibility purposes, most of the price values cannot be abbreviated, so a one-\/minute candle of \char`\"{}\+EUR/\+USD\char`\"{} bid price shall be specified with \char`\"{}\+EUR/\+USD\{=m,price=bid\}\char`\"{} candle symbol string. However, the \mbox{\hyperlink{structCandlePrice_aac615d755dcc3fef383c593c932ede33}{Candle\+Price\+::\+SETTLEMENT}} can be abbreviated to \char`\"{}s\char`\"{}, so a daily candle on \char`\"{}/\+ES\char`\"{} futures settlement prices can be specified with \char`\"{}/\+ES\{=d,price=s\}\char`\"{} string. 
\item \char`\"{}tho\char`\"{} key with a value of \char`\"{}true\char`\"{} corresponds to \mbox{\hyperlink{structCandleSymbol_a60dee53b86d61315e51f0ce00a7fa6bc}{session}} set to \mbox{\hyperlink{structCandleSession_a0bcf55985ba269b352c4868ddaeb5ae9}{Candle\+Session\+::\+REGULAR}} which limits the candle to trading hours only, so a 133 tick candles on \char`\"{}\+GOOG\char`\"{} base symbol collected over trading hours only can be specified with \char`\"{}\+GOOG\{=133t,tho=true\}\char`\"{} string. Note, that the default daily candles for US equities are special for historical reasons and correspond to the way US equity exchange report their daily summary data. The volume the US equity default daily candle corresponds to the total daily traded volume, while open, high, low, and close correspond to the regular trading hours only. 
\item \char`\"{}a\char`\"{} key corresponds to \mbox{\hyperlink{structCandleSymbol_a8fc1a5cb6364857d7be7ecacada03db8}{alignment}} --- alignment attribute of this symbol. The \mbox{\hyperlink{structCandleAlignment}{Candle\+Alignment}} defines possible values with \mbox{\hyperlink{structCandleAlignment_a079e387c53a40c7660131a528d509851}{Candle\+Alignment\+::\+MIDNIGHT}} being default. The alignment values can be abbreviated to the first letter. So, a 1 hour candle on a symbol \char`\"{}\+AAPL\char`\"{} that starts at the regular trading session at 9\+:30 am ET can be specified with \char`\"{}\+AAPL\{=h,a=s,tho=true\}\char`\"{}. Contrast that to the \char`\"{}\+AAPL\{=h,tho=true\}\char`\"{} candle that is aligned at midnight and thus starts at 9\+:00 am. 
\item \char`\"{}pl\char`\"{} key corresponds to \mbox{\hyperlink{structCandleSymbol_a28e59169e3823aba617d6b04b2984061}{price level}} --- price level attribute of this symbol. The \mbox{\hyperlink{structCandlePriceLevel}{Candle\+Price\+Level}} defines additional axis to split candles within particular price corridor in addition to \mbox{\hyperlink{structCandlePeriod}{Candle\+Period}} attribute with the default value {\ttfamily NaN}. So a one-\/minute candles of \char`\"{}\+AAPL\char`\"{} with price level 0.\+1 shall be specified with \char`\"{}\+AAPL\{=m,pl=0.\+1\}\char`\"{}. 
\end{DoxyItemize}

Keys in the candle symbol are case-\/sensitive, while values are not. The \mbox{\hyperlink{structCandleSymbol_ab4294a03f10c11e642f03c2be68a2cc0}{Candle\+Symbol\+::value\+Of()}} method parses any valid string representation into a candle symbol object. The result of the candle symbol \mbox{\hyperlink{structCandleSymbol_a253362a301d71f3d83f8c51ebbd8aa9f}{Candle\+Symbol\+::to\+String()}} method is always normalized\+: keys are ordered lexicographically, values are in lower-\/case and are abbreviated to their shortest possible form.
