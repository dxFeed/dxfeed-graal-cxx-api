\doxysubsubsection{Detailed Description}
Provides on-\/demand historical tick data replay controls. 

This class is used to seamlessly transition from ordinary real-\/time or delayed data feed to the replay of the tick-\/by-\/tick history of market data behaviour without any changes to the code that consumes and process these market events.

As single \doxylink{struct_on_demand_service}{On\+Demand\+Service} instance is conceptually associated with each \doxylink{struct_d_x_endpoint}{DXEndpoint} endpoint instance with the role of \doxylink{struct_d_x_endpoint_a4257e04caaa1ff898a60ece339e02810ae352a08d4fba713e5ae1e538bf27258c}{FEED} or \doxylink{struct_d_x_endpoint_a4257e04caaa1ff898a60ece339e02810a4e0e3dbd8f06b71f54c84c97fea30e22}{ON\+\_\+\+DEMAND\+\_\+\+FEED}. You can retrieve \doxylink{struct_on_demand_service}{On\+Demand\+Service} instance that is associated with a given \doxylink{struct_d_x_endpoint}{DXEndpoint} using \doxylink{struct_on_demand_service_ac5889b10015c99e168ea5ffd420b5d9e}{get\+Instance(endpoint)} method.

For example, \doxylink{struct_on_demand_service}{On\+Demand\+Service} instance for a default \doxylink{struct_d_x_feed}{DXFeed} that is featured in the documentation of \doxylink{struct_d_x_feed}{DXFeed} class and is retrieved via \doxylink{struct_d_x_feed_ac03322a8e22a52beebd6268fc8a167ee}{DXFeed\+::get\+Instance()} method can be acquired with the following code\+: 
\begin{DoxyPre}{\ttfamily 
\doxylink{struct_d_x_endpoint}{auto} endpoint = \doxylink{struct_d_x_endpoint_a9ee308e31ef33e66ec8b8b83b9cff4de}{DXEndpoint::getInstance()};
\doxylink{struct_on_demand_service}{auto} onDemand = \doxylink{struct_on_demand_service}{OnDemandService}::\doxylink{struct_on_demand_service_ac5889b10015c99e168ea5ffd420b5d9e}{getInstance}(endpoint);
}\end{DoxyPre}


This instance can be used for on-\/demand historical tick data replay only when endpoint is connected to the on-\/demand data provider and the appropriate credentials are provided. \doxylink{struct_on_demand_service_a24e717abdfbba919754d40655d7e46e8}{is\+Replay\+Supported} returns {\ttfamily true} when it is so.

On-\/demand replay is started with \doxylink{struct_on_demand_service_a6bf7aa5bc8a3c3c2bc60376b7a8b7bb6}{replay} method that takes time as a parameter. \doxylink{struct_d_x_feed}{DXFeed} is then disconnected from other data providers, pre-\/buffers historical tick data for replay, and starts replay of the data as if it was coming from the data feeds now. All the usual APIs that are part of \doxylink{struct_d_x_feed}{DXFeed} like \doxylink{class_d_x_feed_subscription}{subscription} and various dx\+Feed models can be used normally as with the ordinary real-\/time or delayed data feed.

Replay speed can be changed on the fly with \doxylink{struct_on_demand_service_ad778940f150ac51772b103f900a1de8f}{set\+Speed} method or it can be set initially when starting replay with a two-\/argument version of \doxylink{struct_on_demand_service_a6bf7aa5bc8a3c3c2bc60376b7a8b7bb6}{replay} method. \doxylink{struct_on_demand_service_a75442709db7814fc8344a08b3c58c4d8}{pause} method is the same as \doxylink{struct_on_demand_service_ad778940f150ac51772b103f900a1de8f}{set\+Speed(0)}.

\doxylink{struct_on_demand_service_a6bdef0502b1c6b0026dadd21d8553835}{stop\+And\+Resume} method stops data replay and resumes ordinary real-\/time or delayed data feed that was used before \doxylink{struct_on_demand_service_a6bf7aa5bc8a3c3c2bc60376b7a8b7bb6}{replay} method was invoked. Endpoints with a role of \doxylink{struct_d_x_endpoint_a4257e04caaa1ff898a60ece339e02810a4e0e3dbd8f06b71f54c84c97fea30e22}{ON\+\_\+\+DEMAND\+\_\+\+FEED} do not have an ordinary feed (they are essentially on-\/demand only) and thus \doxylink{struct_on_demand_service_a6bdef0502b1c6b0026dadd21d8553835}{stop\+And\+Resume} method works like \doxylink{struct_on_demand_service_a8d657647f6b2f0b267dd30d7dc384bd7}{stop\+And\+Clear} for them.

\doxysubsubsubsubsection*{State and change notifications}

On-\/demand historical tick data replay state can be queried with \doxylink{struct_on_demand_service_a24e717abdfbba919754d40655d7e46e8}{is\+Replay\+Supported}, \doxylink{struct_on_demand_service_a249178c137cbf45f3b63918a903ecedb}{is\+Replay}, \doxylink{struct_on_demand_service_a7cf67f7bd15f591aaa4ef19cccea8141}{get\+Speed}, and \doxylink{struct_on_demand_service_a2449b333a442c5e9189471b10b0fee2a}{get\+Time} methods.

\doxysubsubsubsubsection*{Threads and locks}

This class is thread-\/safe and can be used concurrently from multiple threads without external synchronization.

To stop on-\/demand historical data replay without resuming ordinary data feed use either \doxylink{struct_on_demand_service_a75442709db7814fc8344a08b3c58c4d8}{On\+Demand\+Service\+::pause()} to keep data snapshot or \doxylink{struct_on_demand_service_a8d657647f6b2f0b267dd30d7dc384bd7}{On\+Demand\+Service\+::stop\+And\+Clear()} to clear data.
