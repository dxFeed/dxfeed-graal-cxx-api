\doxysubsubsection{Detailed Description}
Base class for common fields of \doxylink{class_order}{Order}, \doxylink{class_analytic_order}{Analytic\+Order} and \doxylink{class_spread_order}{Spread\+Order} events. 

\doxylink{class_order}{Order} events represent a snapshot for a full available market depth for a symbol. The collection of order events of a symbol represents the most recent information that is available about orders on the market at any given moment of time.

\doxylink{class_order}{Order} event represents market depth for a {\bfseries{specific symbol}}.

\doxylink{class_analytic_order}{Analytic\+Order} event represents market depth for a {\bfseries{specific symbol}} extended with an analytic information, for example, whether particular order represent an iceberg or not.

\doxylink{class_spread_order}{Spread\+Order} event represents market depth for {\bfseries{all spreads on a given underlying symbol}}.

\doxylink{class_order}{Order} events arrive from multiple sources for the same market symbol and are distinguished by their \doxylink{class_order_base_a21bc36fb881577927015977bc5f71ec6}{index}. Index is a unique per symbol identifier of the event. It is unique across all the sources of depth information for the symbol. The event with \doxylink{class_order_base_a1a128e82af72d3fcd44a7f3eb8bcfd44}{size} either {\ttfamily 0} or {\ttfamily NaN} is a signal to remove previously received order for the corresponding index. The method \doxylink{class_order_base_afb5bea9f67fcdd11410562ab462b901d}{has\+Size} is a convenient method to test for size presence.

\doxysubsubsubsubsection*{\label{class_underlying_eventFlagsSection}%
\Hypertarget{class_underlying_eventFlagsSection}%
Event flags, transactions and snapshots}

Some order event sources provide a consistent view of the price-\/level or detailed order book. Their updates may incorporate multiple changes to price levels or to individual orders that have to be processed at the same time. The corresponding information is carried in \doxylink{class_order_base_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags} property. The logic behind this property is detailed in \doxylink{struct_indexed_event}{Indexed\+Event} class documentation.

The event \doxylink{class_order_base_a03b77983ab4b799f1c7f27aecc943e57}{source} identifier for an order is a part of the unique event \doxylink{class_order_base_a21bc36fb881577927015977bc5f71ec6}{index}. It occupies highest bits of the \doxylink{class_order_base_a21bc36fb881577927015977bc5f71ec6}{index} (index is not-\/negative). The lowest bits of \doxylink{class_order_base_a21bc36fb881577927015977bc5f71ec6}{index} contain source-\/specific event index which is always zero in an event that is marked with \doxylink{class_event_flag_adc2ec8ed07a2a2efaf2389fe4f7459c6}{Event\+Flag\+::\+SNAPSHOT\+\_\+\+END} bit in \doxylink{class_order_base_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags}.

Note that for an order with \doxylink{class_event_flag_af6f440540c8c371ef1acef7d910ab23d}{Event\+Flag\+::\+REMOVE\+\_\+\+EVENT} bit in \doxylink{class_order_base_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags} it is always the case that \doxylink{class_order_base_a1a128e82af72d3fcd44a7f3eb8bcfd44}{size} is either {\ttfamily 0} or {\ttfamily NaN}, so no additional logic to process this bit is required for orders. Transactions and snapshots may include orders with \doxylink{class_order_base_a1a128e82af72d3fcd44a7f3eb8bcfd44}{size} of {\ttfamily 0}. The filtering that distinguishes those events as removals of orders shall be performed after all transactions and snapshot processing.

Some aggregated feeds (like CME market depth) are mapped into two distinct source ids (one for buy orders and one for sell orders), but updates and transactions may span both. It is important to keep a separate track of transactional state for each source id, but, at the same time, when DXFeed\+Event\+Listener\+::events\+Received() method is called for a list of events, the order book shall be considered complete and consistent only when all events from the given list are processed.
