\doxysubsubsection{Detailed Description}
\doxylink{class_candle}{Candle} event with open, high, low, close prices and other information for a specific period. 

Candles are build with a specified \doxylink{struct_candle_period}{Candle\+Period} using a specified \doxylink{struct_candle_price}{Candle\+Price} type with a data taken from the specified \doxylink{struct_candle_exchange}{Candle\+Exchange} from the specified \doxylink{struct_candle_session}{Candle\+Session} with further details of aggregation provided by \doxylink{struct_candle_alignment}{Candle\+Alignment}.

Event symbol of the candle is represented with \doxylink{struct_candle_symbol}{Candle\+Symbol} class. Since the \`{}\+Candle is a time-\/series event, it is typically subscribed to using DXFeed\+Time\+Series\+Subscription class that handles the necessarily wrapping of the symbol into \doxylink{class_time_series_subscription_symbol}{Time\+Series\+Subscription\+Symbol} to specify a subscription time range.

\doxysubsubsubsubsection*{\label{class_underlying_eventFlagsSection}%
\Hypertarget{class_underlying_eventFlagsSection}%
Event flags, transactions and snapshots}

Some candle sources provide a consistent view of the set of known candles. The corresponding information is carried in \doxylink{class_candle_aec9692d1a464d7526280204185b6f5c5}{event\+Flags} property. The logic behind this property is detailed in \doxylink{struct_indexed_event}{Indexed\+Event} class documentation. Multiple event sources for the same symbol are not supported for candles, thus \doxylink{struct_time_series_event_a3c0a02028f2c42e8908f5bcdda02c0ff}{source} property is always \doxylink{class_indexed_event_source_ab359b82d74ee151707106b66eec0a4e3}{DEFAULT}.

Time\+Series\+Event\+Model class handles all the snapshot and transaction logic and conveniently represents a list current of time-\/series events order by their \doxylink{class_candle_a2a229f6b56a57a9277bac05388466ebc}{time}. It relies on the code of Abstract\+Indexed\+Event\+Model to handle this logic. Use the source code of Abstract\+Indexed\+Event\+Model for clarification on transactions and snapshot logic.

\doxysubsubsubsubsection*{Publishing Candles}

Publishing of candle events follows the general rules explained in \doxylink{struct_time_series_event}{Time\+Series\+Event} class documentation.

\doxysubsubsubsubsection*{Implementation details}

This event is implemented on top of QDS record {\ttfamily Trade\+History} for tick candles with \doxylink{struct_candle_period_a2934f6797a1c4aeb8ff49560370f0550}{Candle\+Period\+::\+TICK}, records {\ttfamily \doxylink{class_trade}{Trade}.\texorpdfstring{$<$}{<}period\texorpdfstring{$>$}{>}} for a certain set of popular periods, and QDS record {\ttfamily \doxylink{class_candle}{Candle}} for arbitrary custom periods, with a set of {\ttfamily \doxylink{class_candle}{Candle}\{\texorpdfstring{$<$}{<}attributes\texorpdfstring{$>$}{>}\}} records for a popular combinations of custom candle symbol attributes like \doxylink{struct_candle_price}{Candle\+Price} for an efficient support of bid-\/ask charting.
