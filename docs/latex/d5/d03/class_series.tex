\doxysubsubsection{Detailed Description}
\doxylink{class_series}{Series} event is a snapshot of computed values that are available for all option series for a given underlying symbol based on the option prices on the market. 

It represents the most recent information that is available about the corresponding values on the market at any given moment of time.

\doxylink{class_series}{Series} is an \doxylink{struct_indexed_event}{Indexed\+Event} with multiple instances of event available for each underlying symbol. Each series event instance corresponds to an Option\+Series of the corresponding underlying. The correspondence between a series event instance and an Option\+Series is established via \doxylink{class_series_ad9d84082306a664c5c1d7f0ca8b7fc6e}{expiration} property. If case where there are multiple series at the same expiration day id, then series events are are ordered by their \doxylink{class_series_a7813d76291d673ec40f1dbc4e91a957a}{index} in the same order as the corresponding Option\+Series are ordered by their attributes.

Event flags, transactions and snapshots}

\doxylink{class_series}{Series} data source provides a consistent view of the set of known series. The corresponding information is carried in \doxylink{class_series_aa7a2cf69f440c96e959215d91b74be18}{event\+Flags} property. The logic behind this property is detailed in \doxylink{struct_indexed_event}{Indexed\+Event} class documentation. Multiple event sources for the same symbol are not supported for series, thus \doxylink{class_series_aae1b05fd27e0987ff366315857e5c5f2}{source} property is always \doxylink{class_indexed_event_source_ab359b82d74ee151707106b66eec0a4e3}{DEFAULT}.

Indexed\+Event\+Model class handles all the snapshot and transaction logic and conveniently represents a list current of events.

This event is implemented on top of QDS records {\ttfamily \doxylink{class_series}{Series}}.
