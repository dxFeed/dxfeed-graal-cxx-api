\doxysubsubsection{Detailed Description}
\mbox{\hyperlink{classOrder}{Order}} event is a snapshot for a full available market depth for a symbol. 

The collection of order events of a symbol represents the most recent information that is available about orders on the market at any given moment of time. \mbox{\hyperlink{classOrder}{Order}} events give information on several levels of details, called scopes -\/ see \mbox{\hyperlink{structScope}{Scope}}. \mbox{\hyperlink{structScope}{Scope}} of an order is available via \mbox{\hyperlink{classOrderBase_a1dbb8c850e9bfe281d3e74e59ff62d71}{scope}} property.

\mbox{\hyperlink{classOrder}{Order}} events arrive from multiple sources for the same market symbol and are distinguished by their \mbox{\hyperlink{classOrderBase_a21bc36fb881577927015977bc5f71ec6}{index}}. Index is a unique per symbol identifier of the event. It is unique across all the sources of depth information for the symbol. The event with \mbox{\hyperlink{classOrderBase_a1a128e82af72d3fcd44a7f3eb8bcfd44}{size}} either {\ttfamily 0} or {\ttfamily NaN} is a signal to remove previously received order for the corresponding index. The method \mbox{\hyperlink{classOrderBase_afb5bea9f67fcdd11410562ab462b901d}{Order\+::has\+Size()}} is a convenient method to test for size presence.

Events from finer-\/grained \mbox{\hyperlink{structScope}{Scope}} of detail give more information and include events from coarse-\/grained \mbox{\hyperlink{structScope}{Scope}} of detail. For a consistent representation of the market depth to the end-\/user, order events must be filtered to remove coarse-\/grained events that are superseded by the finer-\/grained events.

\doxysubparagraph*{\label{_eventFlagsSection}%
Event flags, transactions and snapshots}

Some order event sources provide a consistent view of the price-\/level or detailed order book. Their updates may incorporate multiple changes to price levels or to individual orders that have to be processed at the same time. The corresponding information is carried in \mbox{\hyperlink{classOrderBase_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags}} property.

See {\ttfamily Event Flags} section of \mbox{\hyperlink{classOrderBase}{Order\+Base}} class documentation for details.

The composite quotes with \mbox{\hyperlink{structScope_ad5b8601448fc3bf4b6406a5e26cd3dac}{Scope\+::\+COMPOSITE}} and regional quotes with \mbox{\hyperlink{structScope_ad5ba143a0570faeb8c5c041a8ca93b8f}{Scope\+::\+REGIONAL}} come individually from different venues and are not related to each other in any transactional way. The result of \mbox{\hyperlink{classOrderBase_af5e6a4420f059a8e46e664dafebb6ea4}{Order\+::get\+Event\+Flags}} method for them is always zero.

\doxysubparagraph*{Publishing order books}

When publishing an order event with \mbox{\hyperlink{structDXPublisher_ae6e8cb6b370ea60f1ad5cb80cea2df30}{DXPublisher\+::publish\+Events()}} method, least significant 32 bits of order \mbox{\hyperlink{classOrderBase_a21bc36fb881577927015977bc5f71ec6}{index}} must be in a range of from 0 to {\ttfamily std\+::numeric\+\_\+limits$<$std\+::int32\+\_\+t$>$\+::max()} inclusive. Use \mbox{\hyperlink{classOrderBase_a3451d2eb5655cd3c902c78e3368ecfe3}{Order\+::set\+Source()}} method after \mbox{\hyperlink{classOrderBase_a1a7ce2fb19329a2f8450110cb58b9ac0}{Order\+::set\+Index()}} to properly include source identifier into the index.

A snapshot has to be published in the {\itshape descending} order of \mbox{\hyperlink{classOrderBase_a21bc36fb881577927015977bc5f71ec6}{index}}, starting with an event with the largest index and marking it with \mbox{\hyperlink{classEventFlag_ab1246c3963485123672b05d5b9dbdfec}{Event\+Flag\+::\+SNAPSHOT\+\_\+\+BEGIN}} bit in \mbox{\hyperlink{classOrderBase_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags}}, and finishing the snapshot with an event that has zero 32 least significant bits of index. \mbox{\hyperlink{classEventFlag_adc2ec8ed07a2a2efaf2389fe4f7459c6}{Event\+Flag\+::\+SNAPSHOT\+\_\+\+END}} bit in \mbox{\hyperlink{classOrderBase_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags}} is optional during publishing. It will be properly set on receiving end anyway.

\doxysubparagraph*{Limitations}

This event type cannot be used with DXFeed\+::get\+Last\+Event() method.

\doxysubparagraph*{\label{_fobSection}%
Full \mbox{\hyperlink{classOrder}{Order}} Book Support}

Some feeds provide support for \char`\"{}\+Full Order Book\char`\"{} (FOB) where additional fields will be available\+: 
\begin{DoxyItemize}
\item \mbox{\hyperlink{classOrderBase_a7efe9afce45e642c1b0ecc3387b18c67}{action}} -\/ event business meaning (see \mbox{\hyperlink{structOrderAction}{Order\+Action}} for more details) 
\item \mbox{\hyperlink{classOrderBase_af885b39d4cc4628cfc4dcffb9eadf3c3}{action\+Time}} -\/ time of the last action 
\item \mbox{\hyperlink{classOrderBase_aaa261f02ab35ec9bd249be19e5a86845}{order\+Id}} -\/ ID of this order 
\item \mbox{\hyperlink{classOrderBase_a2770d970f2c61e40475e1031f6a0ed2f}{aux\+Order\+Id}} -\/ additional ID for this order 
\item \mbox{\hyperlink{classOrderBase_afcb177faae74576dbd47dd96b10989d3}{trade\+Id}} -\/ trade (order execution) ID 
\item \mbox{\hyperlink{classOrderBase_a11170b380d9cf0d2f16bbf11c32424a8}{trade\+Price}} -\/ price of the trade 
\item \mbox{\hyperlink{classOrderBase_a8db15a6d8dfae74414f85e2477fecab1}{trade\+Size}} -\/ size of the trade 
\end{DoxyItemize}

\doxysubparagraph*{Implementation details}

This event is implemented on top of QDS record {\ttfamily \mbox{\hyperlink{classQuote}{Quote}}} for composite quotes with \mbox{\hyperlink{structScope_ad5b8601448fc3bf4b6406a5e26cd3dac}{Scope\+::\+COMPOSITE}}, record {\ttfamily \mbox{\hyperlink{classQuote}{Quote}}\&X} for regional exchange best quotes with \mbox{\hyperlink{structScope_ad5ba143a0570faeb8c5c041a8ca93b8f}{Scope\+::\+REGIONAL}}, record {\ttfamily Market\+Maker} for market-\/maker quotes and futures price level aggregates with \mbox{\hyperlink{structScope_a93aa5fc2d20e8f6a513e64cc5b906113}{Scope\+::\+AGGREGATE}}, record {\ttfamily \mbox{\hyperlink{classOrder}{Order}}} for the most fine-\/grained \mbox{\hyperlink{structScope_a9d2f3400ab0b7c487262f11d31b8cc3e}{Scope\+::\+ORDER}} with zero source id, and records {\ttfamily \mbox{\hyperlink{classOrder}{Order}}\#$<$source-\/id$>$} for specific source ids, where {\ttfamily $<$source-\/id$>$} is up to 4 ASCII characters with a mnemonic for the source like \char`\"{}\+NTV\char`\"{}.
