\doxysubsubsection{Detailed Description}
Time and Sale represents a trade or other market event with price, like market open/close price, etc. 

Time and Sales are intended to provide information about trades {\bfseries{in a continuous time slice}} (unlike \doxylink{class_trade}{Trade} events which are supposed to provide snapshot about the {\bfseries{current last}} trade).

Time and Sale events have unique \doxylink{class_time_and_sale_a1eb5db17d7be1d390e30483f7a41297d}{index} which can be used for later correction/cancellation processing.

Some time and sale sources provide a consistent view of the set of known time and sales for a given time range when used with DXFeed\+Time\+Series\+Subscription\}. The corresponding information is carried in \doxylink{class_time_and_sale_aa9fd9d054986905ec24b959114ef5a4e}{event\+Flags} property. The logic behind this property is detailed in \doxylink{struct_indexed_event}{Indexed\+Event} class documentation. Multiple event sources for the same symbol are not supported for time and sales, thus \doxylink{class_time_and_sale_a7b41b9554f0fbf6b763cdef4d9fb800a}{source} property is always \doxylink{class_indexed_event_source_ab359b82d74ee151707106b66eec0a4e3}{DEFAULT}.

Regular subscription via \doxylink{class_d_x_feed_subscription}{DXFeed\+Subscription} produces a stream of time and sale events as they happen and their \doxylink{class_time_and_sale_aa9fd9d054986905ec24b959114ef5a4e}{event\+Flags} are always zero.

Publishing of time and sales events follows the general rules explained in \doxylink{struct_time_series_event}{Time\+Series\+Event} class documentation.

This event is implemented on top of QDS record {\ttfamily \doxylink{class_time_and_sale}{Time\+And\+Sale}}.
