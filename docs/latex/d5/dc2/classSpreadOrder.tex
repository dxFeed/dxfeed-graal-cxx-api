\doxysubsubsection{Detailed Description}
Spread order event is a snapshot for a full available market depth for all spreads on a given underlying symbol. 

The collection of spread order events of a symbol represents the most recent information that is available about spread orders on the market at any given moment of time.

Spread order is similar to a regular \mbox{\hyperlink{classOrder}{Order}}, but it has a \mbox{\hyperlink{classSpreadOrder_abc3e79f1245a8707663610e0933efb0c}{spread\+Symbol}} property that contains the symbol of the actual spread that is being represented by spread order object. \mbox{\hyperlink{structMarketEvent_ab45c1e474a01b8641bd97986930f531d}{event\+Symbol}} property contains the underlying symbol that was used in subscription.

Like regular orders, spread order events arrive from multiple sources for the same market symbol and are distinguished by their \mbox{\hyperlink{classOrderBase_a21bc36fb881577927015977bc5f71ec6}{index}}. It is unique across all the sources of depth information for the symbol. The event with \mbox{\hyperlink{classOrderBase_a1a128e82af72d3fcd44a7f3eb8bcfd44}{size}} either {\ttfamily 0} or {\ttfamily NaN} is a signal to remove previously received order for the corresponding index. The method \mbox{\hyperlink{classOrderBase_afb5bea9f67fcdd11410562ab462b901d}{has\+Size}} is a convenient method to test for size presence.

\doxysubparagraph*{\label{_eventFlagsSection}%
Event flags, transactions and snapshots}

Some spread order event sources provide a consistent view of the price-\/level or detailed order book. Their updates may incorporate multiple changes to price levels or to individual orders that have to be processed at the same time. The corresponding information is carried in \mbox{\hyperlink{classOrderBase_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags}} property.

See {\ttfamily Event Flags} section of \mbox{\hyperlink{classOrderBase}{Order\+Base}} class documentation for details.

Indexed\+Event\+Model class handles all the snapshot and transaction logic and conveniently represents a list current of events. It relies on the code of Abstract\+Indexed\+Event\+Model to handle this logic. Use the source code of Abstract\+Indexed\+Event\+Model for clarification on transactions and snapshot logic.

\doxysubparagraph*{Publishing order books}

When publishing an order event with \mbox{\hyperlink{structDXPublisher_ae6e8cb6b370ea60f1ad5cb80cea2df30}{DXPublisher\+::publish\+Events()}} method, least significant 32 bits of order \mbox{\hyperlink{classOrderBase_a21bc36fb881577927015977bc5f71ec6}{index}} must be in a range of from 0 to {\ttfamily std\+::numeric\+\_\+limits$<$std\+::int32\+\_\+t$>$\+::max()} inclusive. Use \mbox{\hyperlink{classOrderBase_a3451d2eb5655cd3c902c78e3368ecfe3}{set\+Source}} method after \mbox{\hyperlink{classOrderBase_a1a7ce2fb19329a2f8450110cb58b9ac0}{set\+Index}} to properly include source identifier into the index.

A snapshot has to be published in the {\itshape descending} order of \mbox{\hyperlink{classOrderBase_a21bc36fb881577927015977bc5f71ec6}{index}}, starting with an event with the largest index and marking it with \mbox{\hyperlink{structIndexedEvent_ac87eb588da470259ee75b490dbcaabf7}{Spread\+Order\+::\+SNAPSHOT\+\_\+\+BEGIN}} bit in \mbox{\hyperlink{classOrderBase_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags}}, and finishing the snapshot with an event that has zero 32 least significant bits of index. \mbox{\hyperlink{structIndexedEvent_a4604235ec9a455bea7c80d1b71546bf9}{Spread\+Order\+::\+SNAPSHOT\+\_\+\+END}} bit in \mbox{\hyperlink{classOrderBase_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags}} is optional during publishing. It will be properly set on receiving end anyway.

\doxysubparagraph*{Limitations}

This event type cannot be used with DXFeed\+::get\+Last\+Event() method.

\doxysubparagraph*{\label{_fobSection}%
Full \mbox{\hyperlink{classOrder}{Order}} Book Support}

Some feeds provide support for \char`\"{}\+Full Order Book\char`\"{} (FOB) where additional fields will be available\+: 
\begin{DoxyItemize}
\item \mbox{\hyperlink{classOrderBase_a7efe9afce45e642c1b0ecc3387b18c67}{action}} -\/ event business meaning (see \mbox{\hyperlink{structOrderAction}{Order\+Action}} for more details) 
\item \mbox{\hyperlink{classOrderBase_af885b39d4cc4628cfc4dcffb9eadf3c3}{action\+Time}} -\/ time of the last action 
\item \mbox{\hyperlink{classOrderBase_aaa261f02ab35ec9bd249be19e5a86845}{order\+Id}} -\/ ID of this order 
\item \mbox{\hyperlink{classOrderBase_a2770d970f2c61e40475e1031f6a0ed2f}{aux\+Order\+Id}} -\/ additional ID for this order 
\item \mbox{\hyperlink{classOrderBase_afcb177faae74576dbd47dd96b10989d3}{trade\+Id}} -\/ trade (order execution) ID 
\item \mbox{\hyperlink{classOrderBase_a11170b380d9cf0d2f16bbf11c32424a8}{trade\+Price}} -\/ price of the trade 
\item \mbox{\hyperlink{classOrderBase_a8db15a6d8dfae74414f85e2477fecab1}{trade\+Size}} -\/ size of the trade 
\end{DoxyItemize}

\doxysubparagraph*{Implementation details}

This event is implemented on top of QDS records {\ttfamily \mbox{\hyperlink{classSpreadOrder}{Spread\+Order}}\#$<$source-\/id$>$}, where {\ttfamily $<$source-\/id$>$} is up to 4 ASCII characters with a mnemonic for the source like \char`\"{}\+ISE\char`\"{}.
