\doxysubsubsection{Detailed Description}
Provides API for publishing of events to local or remote \doxylink{structdxfcpp_1_1_d_x_feed}{DXFeed}. 

\doxyparagraph*{Sample usage}

This section gives sample usage scenarios.

\doxysubparagraph*{Default singleton instance}

There is a singleton instance of the publisher that is returned by \doxylink{structdxfcpp_1_1_d_x_publisher_a8c124e524c9918e23f7a89f26c09584f}{DXPublisher\+::get\+Instance()} method. It is created on the first use with default configuration properties that are explained in detail in documentation for \doxylink{structdxfcpp_1_1_d_x_endpoint}{DXEndpoint} class in the "{}\+Default properties"{} section. In particular, you can provide a default address to connect using \textquotesingle{}\doxylink{structdxfcpp_1_1_d_x_endpoint_a7494f59e150133d36b87c0184c2179eb}{dxpublisher.address}\textquotesingle{} system property or by putting it into \textquotesingle{}\doxylink{structdxfcpp_1_1_d_x_endpoint_a200a1641fd464f782b6715fc7d1c1d69}{dxpublisher.properties}\textquotesingle{} file in the current directory.

Publishes events to the corresponding feed. 

If the \doxylink{structdxfcpp_1_1_d_x_endpoint}{endpoint} of this publisher has \doxylink{structdxfcpp_1_1_d_x_endpoint_a8532cb77e64847d745d727a6ef5f2751}{role} of \doxylink{structdxfcpp_1_1_d_x_endpoint_ac5cd7cd6eed2fdaf9daf96d4e0cf9f2da02c25714ac6e2973dfb530a7520c80ae}{DXEndpoint\+::\+Role\+::\+PUBLISHER} and it is connected, the published events will be delivered to the remote endpoints. Local \doxylink{structdxfcpp_1_1_d_x_endpoint_aadf6d4f1eeb73b91c4f7f1deca4bb61f}{feed} will always receive published events.

This method serializes all events into internal representation, so that the instance of the collection as well as the instances of events can be reused after invocation of this method returns.

\doxylink{structdxfcpp_1_1_d_x_feed}{DXFeed} instances that are connected to this publisher either locally or via network receive published events if and only if they are subscribed to the corresponding symbols, or they are subscribed via \doxylink{structdxfcpp_1_1_wildcard_symbol_a7c4084f144af571c8ac73b68c49dbc5c}{Wildcard\+Symbol\+::\+ALL}, or, in case of \doxylink{structdxfcpp_1_1_time_series_event}{Time\+Series\+Event} type, they are subscribed via DXFeed\+Time\+Series\+Subscription for the corresponding symbol and time frame.

Published events are not stored and get immediately lost if there is no subscription. Last published events of \doxylink{structdxfcpp_1_1_lasting_event}{Lasting\+Event} types are cached as long as subscription to them is maintained via a specific event symbol (\doxylink{structdxfcpp_1_1_wildcard_symbol_a7c4084f144af571c8ac73b68c49dbc5c}{Wildcard\+Symbol\+::\+ALL} does not count) and the cache is discarded as soon as subscription disappears.
