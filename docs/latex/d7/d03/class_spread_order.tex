\doxysubsubsection{Detailed Description}
Spread order event is a snapshot for a full available market depth for all spreads on a given underlying symbol. 

The collection of spread order events of a symbol represents the most recent information that is available about spread orders on the market at any given moment of time.

Spread order is similar to a regular \doxylink{class_order}{Order}, but it has a \doxylink{class_spread_order_a0b41f19add3468de11a133c19065e20a}{spread\+Symbol} property that contains the symbol of the actual spread that is being represented by spread order object. \doxylink{struct_market_event_a6f4f9f4bdb7ef23cda9d5a57640a8db3}{event\+Symbol} property contains the underlying symbol that was used in subscription.

Like regular orders, spread order events arrive from multiple sources for the same market symbol and are distinguished by their \doxylink{class_order_base_a21bc36fb881577927015977bc5f71ec6}{index}. It is unique across all the sources of depth information for the symbol. The event with \doxylink{class_order_base_a1a128e82af72d3fcd44a7f3eb8bcfd44}{size} either {\ttfamily 0} or {\ttfamily NaN} is a signal to remove previously received order for the corresponding index. The method \doxylink{class_order_base_afb5bea9f67fcdd11410562ab462b901d}{has\+Size} is a convenient method to test for size presence.

\doxysubsubsubsubsection*{\label{class_underlying_eventFlagsSection}%
\Hypertarget{class_underlying_eventFlagsSection}%
Event flags, transactions and snapshots}

Some spread order event sources provide a consistent view of the price-\/level or detailed order book. Their updates may incorporate multiple changes to price levels or to individual orders that have to be processed at the same time. The corresponding information is carried in \doxylink{class_order_base_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags} property.

See {\ttfamily Event Flags} section of \doxylink{class_order_base}{Order\+Base} class documentation for details.

Indexed\+Event\+Model class handles all the snapshot and transaction logic and conveniently represents a list current of events. It relies on the code of Abstract\+Indexed\+Event\+Model to handle this logic. Use the source code of Abstract\+Indexed\+Event\+Model for clarification on transactions and snapshot logic.

\doxysubsubsubsubsection*{Publishing order books}

When publishing an order event with \doxylink{struct_d_x_publisher_ae6e8cb6b370ea60f1ad5cb80cea2df30}{DXPublisher\+::publish\+Events()} method, least significant 32 bits of order \doxylink{class_order_base_a21bc36fb881577927015977bc5f71ec6}{index} must be in a range of from 0 to {\ttfamily std\+::numeric\+\_\+limits\texorpdfstring{$<$}{<}std\+::int32\+\_\+t\texorpdfstring{$>$}{>}::max()} inclusive. Use \doxylink{class_order_base_a3451d2eb5655cd3c902c78e3368ecfe3}{set\+Source} method after \doxylink{class_order_base_a1a7ce2fb19329a2f8450110cb58b9ac0}{set\+Index} to properly include source identifier into the index.

A snapshot has to be published in the {\itshape descending} order of \doxylink{class_order_base_a21bc36fb881577927015977bc5f71ec6}{index}, starting with an event with the largest index and marking it with \doxylink{struct_indexed_event_ac87eb588da470259ee75b490dbcaabf7}{Spread\+Order\+::\+SNAPSHOT\+\_\+\+BEGIN} bit in \doxylink{class_order_base_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags}, and finishing the snapshot with an event that has zero 32 least significant bits of index. \doxylink{struct_indexed_event_a4604235ec9a455bea7c80d1b71546bf9}{Spread\+Order\+::\+SNAPSHOT\+\_\+\+END} bit in \doxylink{class_order_base_af5e6a4420f059a8e46e664dafebb6ea4}{event\+Flags} is optional during publishing. It will be properly set on receiving end anyway.

\doxysubsubsubsubsection*{Limitations}

This event type cannot be used with DXFeed\+::get\+Last\+Event() method.
