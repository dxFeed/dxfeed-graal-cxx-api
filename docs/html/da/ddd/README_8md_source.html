6 This package provides access to [dxFeed market data](https://dxfeed.com/market-data/).
7 The library is built as a language-specific wrapper over
8 the [dxFeed Graal Native](https://dxfeed.jfrog.io/artifactory/maven-open/com/dxfeed/graal-native-api/) library,
9 which was compiled with [GraalVM Native Image](https://www.graalvm.org/latest/reference-manual/native-image/)
10 and [dxFeed Java API](https://docs.dxfeed.com/dxfeed/api/overview-summary.html) (our flagman API).

12 :information_source: If you already use [dxFeed C API](https://github.com/dxFeed/dxfeed-c-api), please see
13 the [Overview](#overview) section.<br>

44 ## Overview
45 
46 ### Reasons for the New CXX API Repository
47 
48 The old version of [dxFeed C API](https://github.com/dxFeed/dxfeed-c-api), has
49 several [architectural restrictions](#architectural-restrictions-and-other-limitations-of-the-old-version)
50 that prevent us from providing a state-of-the-art technological solution.
51 
52 ### Benefits of the New Version
53 
54 - :rocket: Increased performance
55 - :milky_way: Wider functionality
56 - :gemini: Identical programming interfaces to our best API
57 - :thumbsup: Higher quality of support and service
58 
59 ### Milestones
60 
61 Feature development has already stopped for the [old version](https://github.com/dxFeed/dxfeed-c-api) of dxFeed C
62 API.
63 
64 The old version will be considered deprecated, and at the end of 2024, we plan to end the service. If you’re already our customer and have difficulty with a future transition, please contact us via our customer portal.
65

67

69 API and how several functionalities are implemented. More examples will follow. The table below shows the sample mapping

71 
72 Our support team on our [customer portal](https://jira.in.devexperts.com/servicedesk/customer/portal/1/create/122) is
73 ready to answer any questions and help with the transition.

85 ### Implementation Details
86 
87 We use [GraalVM Native Image](https://www.graalvm.org/latest/reference-manual/native-image/) technology and specially
88 written code that *wraps* Java methods into native ones
89 to get dynamically linked libraries for different platforms (Linux, macOS, and Windows) based on
90 the [latest Java API package](https://dxfeed.jfrog.io/artifactory/maven-open/com/devexperts/qd/dxfeed-api/).
91 
92 Then, the resulting dynamic link library (dxFeed Graal-native) is used through
93 C [ABI](https://en.wikipedia.org/wiki/Application_binary_interface) (application binary interface),
94 and we write programming interfaces that describe our business model (similar to Java API).
95 
96 As a result, we get a full-featured, similar performance as with Java API.
97 Regardless of the language, writing the final application logic using API calls will be very similar (only the syntax
98 will be amended, *"best practices"*, specific language restrictions).

107 ### Architectural Restrictions and Other Limitations of the Old Version
108 
109 | # | Limitation | How It’s Solved in the New Version |
110 |:-:|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
111 | 1 | Single-threaded architecture limiting throughput | Based on the Java API, each subscription object ([DXFeedSubscription](https://docs.dxfeed.com/dxfeed/api/com/dxfeed/api/DXFeedSubscription.html)) *can* run on its own thread |
112 | 2 | User code in event callbacks (for example, [dxf_event_listener_t](https://docs.dxfeed.com/c-api/group__event-data-structures-event-subscription-stuff.html#gac8bcb70cd4c8857f286f4be65e9522c6)) is executed in the socket read thread, which can significantly reduce throughput | Socket processing threads and callback threads are separated |
113 | 3 | In event callbacks, one market event type and one data portion always arrive (excluding snapshot subscription), which increases the load on the CPU with a large amount of incoming data | Event callbacks can receive different market event types, and more than one by batch |
114 | 4 | It’s impossible to subscribe to data without getting [regionals](https://kb.dxfeed.com/en/data-model/exchange-codes.html) (if it is available for the market event) or only for a certain regional | `subscription->addSymbols({"AAPL"});` - [composite](https://kb.dxfeed.com/en/data-model/qd-model-of-market-events.html#quote-47603)<br>`subscription->addSymbols({"AAPL&Q"});` - [regional](https://kb.dxfeed.com/en/data-model/qd-model-of-market-events.html#quote-x--regional-quote-) |
115 | 5 | It’s impossible to subscribe to `Order` event (excluding snapshot subscription) without getting: all [sources](https://kb.dxfeed.com/en/data-model/qd-model-of-market-events.html#order-x), `Order` by `Quote` (including regionals), Order by [MarketMaker](https://kb.dxfeed.com/en/data-model/qd-model-of-market-events.html#marketmaker-47603) | `subscription->addSymbols(IndexedEventSubscriptionSymbol::create("AAPL", OrderSource::NTV));` - [OrderSource](https://github.com/dxFeed/dxfeed-graal-cxx-api/blob/main/include/dxfeed_graal_native_cpp_api/event/market/OrderSource.hpp) determines which data is being subscribed to |
116 | 6 | Data is mixed up when creating two subscriptions (regular and time series) for the same market event type. Both regular and time series data go to both subscriptions | Each subscription instance receives only the data requested |
117 | 7 | Each subsequent request for the same symbol set in a subscription instance overwrites the existing one in another subscription instance | Subscription instances and the data they receive are independent of each other |
118 | 8 | Removing a symbol from one subscription instance caused it to be removed from all others | Subscription instances and the data they receive are independent of each other |
119 | 9 | Incorrect behavior when reading from a file (if a market event in the file hasn\'t been subscribed to). Reading from a file always occurs at maximum speed. The supported format is binary only | `endpoint->connect("file:tape.txt[format=text]");` - processing a text file with at it's "real" speed by timestamps<br>`endpoint->connect("file:tape.bin[format=binary,speed=max]");` - processing a binary file with max speed |

190 ## Installation
191 
192 Examples of how to install the library and integrate with the library can be
193 found [here](https://github.com/dxFeed/dxfeed-graal-cxx-api-samples).
194 
195 The API is delivered as a dynamic library (`dxFeedGraalCxxApi.so|dll|dylib`) and a static
196 one (`dxFeedGraalCxxApi_static.a|lib`).

198 use `RPATH`, `LD_LIBRARY_PATH`).

200 ## Usage
201 
202 ### How to connect to QD endpoint
203 
204 To use the dynamic library define the `DXFCPP_USE_DLLS` preprocessor directive.
205 
206 ```cpp
207 #include <iostream>
208 #include <dxfeed_graal_cpp_api/api.hpp>
209 
210 int main() {
211  using namespace dxfcpp;
212 
213  // For token-based authorization, use the following address format:
214  // "demo.dxfeed.com:7300[login=entitle:token]"
215  auto endpoint = DXEndpoint::newBuilder()
216  ->withProperty("dxfeed.address", "demo.dxfeed.com:7300")
217  ->build();
218 
219  // Another way to connect the endpoint.
220  // auto endpoint = DXEndpoint::create()->connect("demo.dxfeed.com:7300");
221 
222  auto subscription = endpoint->getFeed()->createSubscription(Quote::TYPE);
223 
224  subscription->addEventListener([](auto&& events) {
225  for (auto&& e : events) {
226  std::cout << e << "\n";
227  }
228  });
229 
230  subscription->addSymbols({"AAPL"});
231 
232  std::cin.get();
233 }
234 ```

256 ### How to connect to dxLink
257 
258 ```cpp
259 #include <iostream>
260 #include <dxfeed_graal_cpp_api/api.hpp>
261 
262 int main() {
263  using namespace dxfcpp;
264 
265  // Enable experimental feature.
266  System::setProperty("dxfeed.experimental.dxlink.enable", "true");
267  // Set scheme for dxLink.
268  System::setProperty("scheme", "ext:opt:sysprops,resource:dxlink.xml");
269 
270  // For token-based authorization, use the following address format:
271  // "dxlink:wss://demo.dxfeed.com/dxlink-ws[login=dxlink:token]"
272  auto endpoint = DXEndpoint::newBuilder()
273  ->withProperty("dxfeed.address", "dxlink:wss://demo.dxfeed.com/dxlink-ws")
274  ->build();
275 
276  // Another way to connect the endpoint.
277  // auto endpoint = DXEndpoint::create()->connect("dxlink:wss://demo.dxfeed.com/dxlink-ws");
278 
279  auto subscription = endpoint->getFeed()->createSubscription(Quote::TYPE);
280 
281  subscription->addEventListener([](auto&& events) {
282  for (auto&& e : events) {
283  std::cout << e << "\n";
284  }
285  });
286 
287  subscription->addSymbols({"AAPL"});
288 
289  std::cin.get();
290 }
291 ```

316 ## Tools
317 
318 [Tools](https://github.com/dxFeed/dxfeed-graal-cxx-api/blob/main/tools/Tools/)
319 is a collection of utilities that allow you to subscribe to various market events for the specified symbols. The tools
320 can
321 be downloaded from [Release](https://github.com/dxFeed/dxfeed-graal-cxx-api/releases) (including self-contained
322 versions):
323 
324 * [Connect](https://github.com/dxFeed/dxfeed-graal-cxx-api/blob/main/tools/Tools/src/Connect/ConnectTool.hpp)
325  connects to the specified address(es) and subscribes to the specified events with the specified symbol
326 * [Dump](https://github.com/dxFeed/dxfeed-graal-cxx-api/blob/main/tools/Tools/src/Dump/DumpTool.hpp)
327  dumps all events received from address. This was designed to retrieve data from a file
328 * [PerfTest](https://github.com/dxFeed/dxfeed-graal-cxx-api/blob/main/tools/Tools/src/PerfTest/PerfTestTool.hpp)
329  connects to the specified address(es) and calculates performance counters (events per second, memory usage, CPU usage,
330  etc.)
331 * [LatencyTest](https://github.com/dxFeed/dxfeed-graal-cxx-api/blob/main/tools/Tools/src/LatencyTest/LatencyTestTool.hpp)
332  connects to the specified address(es) and calculates latency
333 * [Qds](https://github.com/dxFeed/dxfeed-graal-cxx-api/blob/main/tools/Tools/src/Qds/QdsTool.hpp)
334  collection of tools ported from the Java qds-tools
