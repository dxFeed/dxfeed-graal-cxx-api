25 /**
26  * Order event is a snapshot for a full available market depth for a symbol.
27  * The collection of order events of a symbol represents the most recent information
28  * that is available about orders on the market at any given moment of time.
29  * Order events give information on several levels of details, called scopes - see Scope.
30  * Scope of an order is available via @ref Order::getScope "scope" property.
31  *
32  * <p> Order events arrive from
33  * multiple sources for the same market symbol and are distinguished by their @ref Order::getIndex "index". Index is a
34  * unique per symbol identifier of the event.
35  * It is unique across all the sources of depth information for the symbol.
36  * The event with @ref Order::getSize() "size" either `0` or `NaN` is a signal to remove previously received order for
37  * the corresponding index. The method Order::hasSize() is a convenient method to test for size presence.
38  *
39  * <p> Events from finer-grained Scope of detail give more information and include events
40  * from coarse-grained Scope of detail. For a consistent representation of the market depth
41  * to the end-user, order events must be filtered to remove coarse-grained events that
42  * are superseded by the finer-grained events.
43  *
44  * <h3><a name="eventFlagsSection">Event flags, transactions and snapshots</a></h3>
45  *
46  * Some order event sources provide a consistent view of the price-level or detailed order book. Their updates
47  * may incorporate multiple changes to price levels or to individual orders that have to be processed at the same time.
48  * The corresponding information is carried in @ref Order::getEventFlags() "eventFlags" property.
49  *
50  * <p> See `Event Flags` section of OrderBase class documentation for details.
51  *
52  * <p> The composite quotes with Scope::COMPOSITE and regional quotes with Scope::REGIONAL come
53  * individually from different venues and are not related to each other in any transactional way. The result of
54  * Order::getEventFlags method for them is always zero.
55  *
56  * <h3>Publishing order books</h3>
57  *
58  * When publishing an order event with DXPublisher::publishEvents() method, least significant 32 bits of
59  * order @ref Order::getIndex() "index" must be in a range of from 0 to `std::numeric_limits<std::int32_t>::max()`
60  * inclusive. Use Order::setSource() method after Order::setIndex() to properly include source identifier into the
61  * index.
62  *
63  * A snapshot has to be published in the <em>descending</em> order of @ref Order::getIndex() "index", starting with
64  * an event with the largest index and marking it with EventFlag::SNAPSHOT_BEGIN bit in @ref Order::getEventFlags()
65  * "eventFlags", and finishing the snapshot with an event that has zero 32 least significant bits of index.
66  * EventFlag::SNAPSHOT_END bit in @ref Order::getEventFlags() "eventFlags" is optional during publishing.
67  * It will be properly set on receiving end anyway.
68  *
69  * <h3>Limitations</h3>
70  *
71  * This event type cannot be used with DXFeed::getLastEvent() method.
72  *
73  * <h3><a name="fobSection">Full Order Book Support</a></h3>
74  *
75  * Some feeds provide support for "Full Order Book" (FOB) where additional fields will be available:
76  * <ul>
77  * <li>@ref Order::getAction() "action" - event business meaning (see OrderAction for more details)</li>
78  * <li>@ref Order::getActionTime() "actionTime" - time of the last action</li>
79  * <li>@ref Order::getOrderId() "orderId" - ID of this order</li>
80  * <li>@ref Order::getAuxOrderId() "auxOrderId" - additional ID for this order</li>
81  * <li>@ref Order::getTradeId() "tradeId" - trade (order execution) ID</li>
82  * <li>@ref Order::getTradePrice() "tradePrice" - price of the trade</li>
83  * <li>@ref Order::getTradeSize() "tradeSize" - size of the trade</li>
84  * </ul>
85  *
86  * <h3>Implementation details</h3>
87  *
88  * This event is implemented on top of QDS record `Quote` for composite quotes with Scope::COMPOSITE,
89  * record `Quote&X` for regional exchange best quotes with Scope::REGIONAL,
90  * record `MarketMaker` for market-maker quotes and futures price level aggregates with Scope::AGGREGATE,
91  * record `Order` for the most fine-grained Scope::ORDER with zero source id,
92  * and records `Order#<source-id>` for specific source ids,
93  * where `<source-id>` is up to 4 ASCII characters with a mnemonic for the source like "NTV".
94  */
