16/**
17 * Represents time-series snapshots of some process that is evolving in time or actual events in some external system
18 * that have an associated time stamp and can be uniquely identified.
19 * For example, TimeAndSale events represent the actual sales that happen on a market exchange at specific time
20 * moments, while Candle events represent snapshots of aggregate information about trading over a specific time period.
21 *
22 * Time-series events can be used with DXFeedTimeSeriesSubscription to receive a time-series history of past events.
23 * Time-series events are conflated based on unique @ref ::getIndex() "index" for each symbol.
24 * Last indexed event for each symbol and index is always delivered to event listeners on subscription, but
25 * intermediate (next-to-last) events for each symbol+index pair are not queued anywhere, they are simply discarded as
26 * stale events.
27 *
28 * Timestamp of an event is available via @ref ::getTime() "time" property with a millisecond precision. Some events
29 * may happen multiple times per millisecond. In this case they have the same @ref ::getTime() "time", but different
30 * @ref ::getIndex() "index". An ordering of @ref ::getIndex() "index" is the same as an ordering
31 * of @ref ::getTime() "time". That is, an collection of time-series events that is ordered
32 * by @ref ::getIndex() "index" is ascending order will be also ordered by @ref ::getTime() "time" in ascending order.
33 *
34 * Time-series events are a more specific subtype of IndexedEvent.
35 * All general documentation and <a href="IndexedEvent.html#eventFlagsSection">Event Flags</a> section, in particular,
36 * applies to time-series events. However, the time-series events never come from multiple sources for the same symbol
37 * and their @ref TimeSeriesEvent::getSource() "source" is always @ref IndexedEventSource::DEFAULT "DEFAULT".
38 *
39 * Unlike a general IndexedEvent that is subscribed to via DXFeedSubscription} using a plain symbol to receive all
40 * events for all indices, time-series events are typically subscribed to using TimeSeriesSubscriptionSymbol class to
41 * specific time range of the subscription. There is a dedicated DXFeedTimeSeriesSubscription class that is designed
42 * to simplify the task of subscribing for time-series events.
43 *
44 * TimeSeriesEventModel class handles all the snapshot and transaction logic and conveniently represents a list of
45 * current time-series events ordered by their @ref TimeSeriesEvent::getTime() "time". It relies on the code
46 * of AbstractIndexedEventModel to handle this logic.
47 * Use the source code of AbstractIndexedEventModel for clarification on transactions and snapshot logic.
48 *
49 * Classes that implement this interface may also implement LastingEvent interface, which makes it possible to use
50 * DXFeed::getLastEvent() method to retrieve the last event for the corresponding symbol.
51 *
52 * <h3>Publishing time-series</h3>
53 *
54 * When publishing time-series event with DXPublisher::publishEvents() method on incoming TimeSeriesSubscriptionSymbol
55 * the snapshot of currently known events for the requested time range has to be published first.
56 *
57 * A snapshot is published in the <em>descending</em> order of @ref TimeSeriesEvent::getIndex() "index" (which is the
58 * same as the descending order of @ref TimeSeriesEvent::getTime() "time"), starting with an event with the largest
59 * index and marking it with TimeSeriesEvent::SNAPSHOT_BEGIN bit in @ref TimeSeriesEvent::getEventFlags "eventFlags".
60 * All other event follow with default, zero
61 * @ref TimeSeriesEvent::getEventFlags() "eventFlags". If there is no actual event at the time that was specified in
62 * subscription
63 * @ref TimeSeriesSubscriptionSymbol::getFromTime() "fromTime" property, then event with the corresponding time
64 * has to be created anyway and published. To distinguish it from the actual event, it has to be marked with
65 * TimeSeriesEvent::REMOVE_EVENT bit in @ref TimeSeriesEvent::getEventFlags() "eventFlags".
66 * TimeSeriesEvent::SNAPSHOT_END bit in this event's
67 * @ref TimeSeriesEvent::getEventFlags() "eventFlags" s optional during publishing. It will be properly set on receiving
68 * end anyway. Note, that publishing any event with time that is below subscription
69 * @ref TimeSeriesSubscriptionSymbol::getFromTime() "fromTime" also works as a legal indicator for the end of the
70 * snapshot.
71 *
72 * Both TimeAndSale and Candle time-series events define a sequence property that is mixed into an
73 * @ref ::getIndex() "index" property. It provides a way to distinguish different events at the same time. For a
74 * snapshot end event the sequence has to be left at its default zero value. It means, that if there is an actual
75 * event to be published at subscription @ref TimeSeriesSubscriptionSymbol::getFromTime() "fromTime" with non-zero
76 * sequence, then generation of an additional snapshot end event with the subscription
77 * @ref TimeSeriesSubscriptionSymbol::getFromTime() "fromTime" and zero sequence is still required.
78 */

83 /**
84 * Returns the source identifier for this event, which is always @ref IndexedEventSource::DEFAULT "DEFAULT" for

88 * time-series events.
89 */

92 }

94 /**
95 * Returns unique per-symbol index of this event.
96 * Event indices are unique within event symbol.
97 * Typically event index for a time series event includes @ref ::getTime() "time" inside.
98 *
99 * <h3>Implementation notes</h3>
100 *
101 * The most common scheme for event indices is to set highest 32 bits of event index
102 * to event timestamp in seconds. The lowest 32 bits are then split as follows.
103 * Bits 22 to 31 encode milliseconds of time stamp, and bits 0 to 21 encode some
104 * kind of a sequence number.
105 *
106 * Ultimately, the scheme for event indices is specific for each even type.
107 * The actual classes for specific event types perform the corresponding encoding.
108 *
109 * @return unique per-symbol index of this event.
110 */
111 std::int64_t getIndex() const noexcept override {
112 return 0;
113 }

116 * Returns unique per-symbol index of this event.
117 *
118 * @return unique per-symbol index of this event.
119 * @deprecated Use ::getIndex()
120 */

122 return getIndex();
123 }

126 * Returns timestamp of the event.
127 * The timestamp is in milliseconds from midnight, January 1, 1970 UTC.
128 *
129 * @return timestamp of the event.
130 */
