18 /**
19  * Represents an indexed collection of up-to-date information about some
20  * condition or state of an external entity that updates in real-time. For example, ::Order represents an order to buy
21  * or to sell some market instrument that is currently active on a market exchange and multiple
22  * orders are active for each symbol at any given moment in time.
23  * ::Candle represent snapshots of aggregate information about trading over a specific time period and there are
24  * multiple periods available.
25  * The ::Candle is also an example of ::TimeSeriesEvent that is a more specific event type.
26  *
27  * Index for each event is available via @ref ::getIndex() "index" property.
28  * Indexed events retain information about multiple events per symbol based on the event index
29  * and are conflated based on the event index. Last indexed event for each symbol and index is always
30  * delivered to event listeners on subscription, but intermediate (next-to-last) events for each
31  * symbol+index pair are not queued anywhere, they are simply discarded as stale events.
32  * More recent events represent an up-to-date information about some external entity.
33  *
34  * Some indexed events, like ::Order, support multiple sources of information for the
35  * same symbol. The event source is available via @ref IndexedEvent::getSource() "source" property.
36  * The value of @ref IndexedEvent::getSource() "source" property is always @ref IndexedEventSource::DEFAULT "DEFAULT"
37  * for time-series events and other singe-sourced events like ::Series, that do not support this feature.
38  *
39  * See also the <a href="./de/d03/classdxfcpp_1_1_event_flag.html#eventFlagsSection">"Event Flags" section</a>
40  */
