26/**
27 * Option Sale event represents a trade or another market event with the price
28 * (for example, market open/close price, etc.) for each option symbol listed under the specified Underlying.
29 * Option Sales are intended to provide information about option trades <b>in a continuous time slice</b> with
30 * the additional metrics, like Option Volatility, Option Delta, and Underlying Price.
31 *
32 * <p>Option Sale events have unique @ref OptionSale::getIndex() "index" which can be used for later
33 * correction/cancellation processing.
34 *
35 * Option sale data source provides a consistent view of the set of known option sales.
36 * The corresponding information is carried in @ref OptionSale::getEventFlags() "eventFlags" property.
37 * The logic behind this property is detailed in IndexedEvent class documentation.
38 * Multiple event sources for the same symbol are not supported for option sale events, thus
39 * @ref OptionSale::getSource() "source" property is always @ref IndexedEventSource::DEFAULT "DEFAULT".
40 *
41 * This event is implemented on top of QDS records `OptionSale`.
42 */
43class DXFCPP_EXPORT OptionSale final : public MarketEvent, public IndexedEvent {
44 friend struct EventMapper;

56 * | | SM | | SS | SE | SB | RE | TX |
57 * +--------+----+----+----+----+----+----+----+
58 * </tt></pre>
59 */
