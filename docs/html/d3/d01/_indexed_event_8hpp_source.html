16/**
17 * Represents an indexed collection of up-to-date information about some
18 * condition or state of an external entity that updates in real-time. For example, ::Order represents an order to buy
19 * or to sell some market instrument that is currently active on a market exchange and multiple
20 * orders are active for each symbol at any given moment in time.
21 * ::Candle represent snapshots of aggregate information about trading over a specific time period and there are
22 * multiple periods available.
23 * The ::Candle is also an example of ::TimeSeriesEvent that is a more specific event type.
24 *
25 * Index for each event is available via @ref ::getIndex() "index" property.
26 * Indexed events retain information about multiple events per symbol based on the event index
27 * and are conflated based on the event index. Last indexed event for each symbol and index is always
28 * delivered to event listeners on subscription, but intermediate (next-to-last) events for each
29 * symbol+index pair are not queued anywhere, they are simply discarded as stale events.
30 * More recent events represent an up-to-date information about some external entity.
31 *
32 * Some indexed events, like ::Order, support multiple sources of information for the
33 * same symbol. The event source is available via @ref IndexedEvent::getSource() "source" property.
34 * The value of @ref IndexedEvent::getSource() "source" property is always @ref IndexedEventSource::DEFAULT "DEFAULT"
35 * for time-series events and other singe-sourced events like ::Series, that do not support this feature.
36 *
37 * See also the <a href="./de/d03/classdxfcpp_1_1_event_flag.html#eventFlagsSection">"Event Flags" section</a>
38 */
39struct DXFCPP_EXPORT IndexedEvent {
