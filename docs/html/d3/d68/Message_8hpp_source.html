23 /**
24  * Message event with application-specific attachment. Messages are never conflated and are delivered to
25  * all connected subscribers. There is no built-in persistence for messages. They are lost when subscribers
26  * are not connected to the message publisher, so they shall be only used for notification purposes in
27  * addition to persistence mechanism.
28  *
29  * <h3>Implementation details</h3>
30  *
31  * This event is implemented on top of QDS record `Message`.
32  */
