28 /**
29  * Candle event with open, high, low, close prices and other information for a specific period.
30  * Candles are build with a specified CandlePeriod using a specified CandlePrice type with a data taken from the
31  * specified CandleExchange from the specified CandleSession with further details of aggregation provided by
32  * CandleAlignment.
33  *
34  * <p> Event symbol of the candle is represented with CandleSymbol class.
35  * Since the `Candle is a time-series event, it is typically subscribed to using DXFeedTimeSeriesSubscription class
36  * that handles the necessarily wrapping of the symbol into TimeSeriesSubscriptionSymbol to specify a subscription
37  * time range.
38  *
39  * <h3><a name="eventFlagsSection">Event flags, transactions and snapshots</a></h3>
40  *
41  * Some candle sources provide a consistent view of the set of known candles.
42  * The corresponding information is carried in @ref Candle::getEventFlags() "eventFlags" property.
43  * The logic behind this property is detailed in IndexedEvent class documentation.
44  * Multiple event sources for the same symbol are not supported for candles, thus @ref Candle::getSource() "source"
45  * property is always @ref IndexedEventSource::DEFAULT "DEFAULT".
46  *
47  * <p>TimeSeriesEventModel class handles all the snapshot and transaction logic and conveniently represents a list
48  * current of time-series events order by their @ref Candle::getTime() "time".
49  * It relies on the code of AbstractIndexedEventModel to handle this logic.
50  * Use the source code of AbstractIndexedEventModel for clarification on transactions and snapshot logic.
51  *
52  * <h3>Publishing Candles</h3>
53  *
54  * Publishing of candle events follows the general rules explained in TimeSeriesEvent class documentation.
55  *
56  * <h3>Implementation details</h3>
57  *
58  * This event is implemented on top of QDS record `TradeHistory` for tick candles with CandlePeriod::TICK,
59  * records `Trade.<period>` for a certain set of popular periods, and QDS record `Candle` for arbitrary custom
60  * periods, with a set of `Candle{<attributes>}` records for a popular combinations of custom candle symbol attributes
61  * like CandlePrice for an efficient support of bid-ask charting.
62  */

330  /**

341  /**
342  * Changes @ref Candle::getSequence() "sequence number" of this event.
343  *
344  * @param sequence the sequence.
345  * @see Candle::getSequence()
346  * @throws std::invalid_argument if sequence is below zero or above ::MAX_SEQUENCE.
347  */
348  void setSequence(std::int32_t sequence) {
349  assert(sequence >= 0 && static_cast<std::uint32_t>(sequence) <= MAX_SEQUENCE);
350 
351  if (sequence < 0 || static_cast<std::uint32_t>(sequence) > MAX_SEQUENCE) {
352  throw std::invalid_argument("Invalid value for argument `sequence`: " + std::to_string(sequence));
353  }
354 
355  data_.index = orOp(andOp(data_.index, ~MAX_SEQUENCE), sequence);
356  }
